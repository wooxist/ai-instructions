# Phase 3.5 재프레이밍 계획

**작성일**: 2025-10-10
**목적**: AI 사고 생태계 프레임워크를 2권의 백본으로 통합
**참조**: `report_kr.md`, `.session/2025-10-10.md` (Session 9)

---

## 개요

2권의 모든 장을 AI 사고 생태계 6계층 구조에 맞춰 재프레이밍합니다:

```
1. 미션 (Mission)
2. 핵심 가치 (Core Values)
3. 비전 (Vision)
   ↓
4. 목표 (Goals) ────────────┐
   ↓                        │
5. 사고 클러스터            │← Part 4-5 핵심 영역
   (Thinking Clusters)      │
   ↓                        │
6. 실행 항목 ───────────────┘
   (Execution Items)
   ↓
피드백 루프 (Feedback Loop)
```

## 11장: 기본 사고 클러스터

### 현재 상태
- **제목**: "11장 2부: 단위 조직 설계"
- **초점**: 아키텍트-워커 팀 구조
- **용어**: 단위 조직, 아키텍트 에이전트, 워커 에이전트

### 재프레이밍 방향

#### 1. 제목 변경
```
현재: "11장 2부: 단위 조직 설계"
새로운: "11장: 기본 사고 클러스터"
```

#### 2. 프레임 전환
| 측면 | 기존 | 새로운 |
|------|------|--------|
| **중심 개념** | 조직 구조 | 사고 단위 |
| **핵심 질문** | "누가 무엇을 하는가?" | "어떻게 생각하고 추론하는가?" |
| **역할 정의** | 아키텍트/워커 에이전트 | 사고 조율자/실행 워커 |
| **목적** | 팀 구성 | 복잡한 목표를 사고 단위로 분해 |

#### 3. 주요 수정 사항

**도입부 재작성**:
- ❌ "전문가 팀 꾸리기" → ✅ "목표를 사고로 분해하기"
- ❌ "단위 조직 패턴" → ✅ "기본 사고 클러스터 패턴"
- AI 사고 생태계에서의 위치 명시 (목표 → 사고 클러스터)

**용어 교체**:
- 단위 조직 → 기본 사고 클러스터
- 아키텍트 에이전트 → 사고 조율자 (Thinking Coordinator)
- 워커 에이전트 → 실행 워커 (Execution Worker)
- "팀 리더" → "사고 프로세스 설계자"

**사례 재구성 (11.2.1: 소셜 미디어 콘텐츠 발행)**:
```markdown
현재 프레임:
- 목표: 팀 구성하여 콘텐츠 발행
- 강조: 역할 분담, 지휘 관계

새로운 프레임:
- 목표: 콘텐츠 발행이라는 목표를 사고 단위로 분해
- 강조: 
  1. 사고 프로세스 설계 (기획 → 작성 → 편집)
  2. 각 단계에서의 추론 방식
  3. 의사결정 지점 (Human-in-the-Loop)
  4. 사고 조율자의 역할: 전체 사고 흐름 관리
```

**새로운 섹션 추가**:
1. "11.1 목표에서 사고 클러스터로" (신규)
   - 전략 목표를 사고 단위로 분해하는 방법
   - 사고 복잡도 평가
   
2. "11.2 사고 조율자의 역할" (기존 내용 재구성)
   - 사고 프로세스 설계
   - 실행 워커 조율
   - 의사결정 지점 관리

3. "11.3 실행 워커의 역할" (기존 내용 재구성)
   - 전문 영역 분석 및 추론
   - 산출물 생성

4. "11.4 사고 흐름 설계" (기존 워크플로우 섹션)
   - 파이프라인 패턴으로 사고 흐름 정의
   - 단계별 추론 구조

#### 4. 다이어그램 업데이트

**조직도 → 사고 구조도**:
```mermaid
현재: 
- 아키텍트 → 워커1, 워커2, 워커3
- 강조: 지휘 관계

새로운:
- 사고 조율자 (사고 프로세스 설계)
  ↓
  실행 워커 그룹 (각 사고 단계 수행)
- 강조: 사고 흐름, 추론 단계
```

**워크플로우 → 사고 흐름도**:
```mermaid
현재: 
- 단계1 → 단계2 → 단계3
- 레이블: "작업", "산출물"

새로운:
- 사고 단계1 → 의사결정 → 사고 단계2 → 사고 단계3
- 레이블: "추론 프로세스", "판단 지점", "사고 산출물"
```

#### 5. 예상 소요 시간
- 도입부 재작성: 30분
- 용어 일괄 교체: 30분
- 사례 재구성: 1시간
- 새로운 섹션 작성: 1.5시간
- 다이어그램 업데이트: 30분
- 검토 및 통합: 30분
**총 예상: 4-5시간**

     * AI: 의존성 분석
