# 작업 가이드

## 현재 작업 확인
PROGRESS.md → "← 작업중" 찾기 → 해당 파일 열기

## 작업 순서
1. Story 파일 Task 확인
2. Task 실행
3. PROGRESS.md 체크
4. Commit: [Q1-P1-S1-S2] 타입: 제목

## 구조
Quarter(3개월)→Phase(1개월)→Sprint(1주)→Story(1일)→Task

## 파일 위치
- 현재: PROGRESS.md 확인
- Story: 00001-Q/stories/phase-01/sprint-01/
- Library: library/tasks/, library/workflows/

## 원칙
- 체크박스: PROGRESS.md만
- 작업: 한 Story씩만
- SSOT: 단일 진실의 원천
- SoC: 파일별 역할 분리
