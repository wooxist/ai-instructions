# Git Commit 규칙

## 형식

```
[Q#-P#-S#-S#] 타입: 제목
```

---

## 타입

| 타입 | 용도 |
|------|------|
| `feat` | 기능 추가 |
| `docs` | 문서 작업 |
| `fix` | 버그 수정 |
| `refactor` | 리팩토링 |
| `chore` | 기타 |

---

## 예시

```bash
[Q1-P1-S2-S3] feat: 사용자 인증 API 구현
[Q1-P2-S1-S5] docs: API 문서 업데이트
[Q2-P1-S3-S2] fix: 데이터 검증 로직 수정
```

---

## 규칙

- Story 완료 시 commit
- 타입은 소문자
- 제목은 명령형
