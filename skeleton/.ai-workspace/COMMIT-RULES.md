# Commit Rules

## 형식

```
[Q#-P#-S#-S#-T#] 타입: 제목
```

## 타입

- **feat**: 새로운 기능 추가
- **docs**: 문서 수정
- **fix**: 버그 수정
- **refactor**: 코드 리팩토링
- **chore**: 기타 작업

## 예시

```
[Q1-P1-S1-S1-T1] feat: 2권 통합 목차 생성
[Q1-P1-S2-S3-T2] docs: 용어 전환 가이드 작성
[Q1-P1-S3-S5-T1] fix: Part 전환 텍스트 수정
```

## 규칙

- Q#: Quarter 번호
- P#: Phase 번호
- S#: Sprint 번호
- S#: Story 번호
- T#: Task 번호
