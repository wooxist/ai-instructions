# 프로젝트 우선순위 관리

## 우선순위 매트릭스

| 프로젝트 | 긴급도 | 중요도 | 상태 | 우선순위 |
|---------|--------|--------|------|---------|
| 00001-Q: AI Instructions | 🔴 높음 | 🔴 높음 | 진행중 | **P1** |
| 00002-Q: Book 요약 | 🟡 중간 | 🔴 높음 | 계획됨 | **P2** |
| 00003-Q: API 시스템 | 🟢 낮음 | 🟡 중간 | 계획됨 | **P3** |

## 작업 전환 규칙

### 일일 작업 배분
- **P1 (00001-Q)**: 70% 시간 할당
- **P2 (00002-Q)**: 20% 시간 할당 (준비/계획)
- **P3 (00003-Q)**: 10% 시간 할당 (리서치)

### Sprint 전환 조건
1. 현재 Sprint의 모든 Story 완료
2. 블로커가 있을 경우 다른 프로젝트로 임시 전환
3. 긴급 요청 시 우선순위 재조정

### 병렬 작업 가능 항목
- 00001-Q Story 실행 중 → 00002-Q 계획 수립 가능
- 00002-Q 분석 중 → 00003-Q 리서치 가능
- 문서화 작업은 모든 프로젝트에서 병렬 가능

## 리스크 관리

### 00001-Q 리스크
- 구조 변경 시 다른 프로젝트 영향
- 완료 지연 시 00002-Q 시작 지연

### 00002-Q 리스크  
- Book 분량이 예상보다 많을 경우
- 요약 품질 기준 미달 시 재작업

### 00003-Q 리스크
- 기술 스택 선정 지연
- 외부 의존성 이슈

## 의사결정 포인트

### 주간 검토 (매주 월요일)
- 각 프로젝트 진행 상황 점검
- 우선순위 재조정 필요성 평가
- 리소스 재배분 결정

### 분기 전환점
- Q1 → Q2: 00001-Q 완료 여부
- Q2 → Q3: 00002-Q 완료 여부
- Q3 → Q4: 00003-Q 완료 여부
