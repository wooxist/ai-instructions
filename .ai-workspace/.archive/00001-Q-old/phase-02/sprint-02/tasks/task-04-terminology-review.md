## 용어집 검토

### 용어집 업데이트 제안

현재 용어 가이드는 매우 잘 작성되어 있으나, 다음 항목을 추가하면 더 완성도가 높아질 것입니다:

#### 추가 권장 용어

**1. AI 사고 생태계 관련**:
```yaml
추가 용어:
  - 미션 (Mission): 조직의 존재 이유
  - 핵심 가치 (Core Values): 의사결정 기준
  - 비전 (Vision): 미래 상태
  - 목표 (Goals): 전략적 목표
  - 실행 항목 (Execution Items): 구체적 작업
```

**2. 사고 프로세스 관련**:
```yaml
추가 용어:
  - 사고 단계 (Thinking Stage): 사고 클러스터 내 단계
  - 사고 흐름 (Thinking Flow): 사고 단계들의 순서
  - 의사결정 지점 (Decision Point): 인간 판단 필요 지점
  - 사고 산출물 (Thinking Output): 사고 단계의 결과
```

**3. 파일 시스템 관련 (Ch 13)**:
```yaml
추가 용어:
  - 사고 상태 (Thinking State): thinking_state.json
  - 사고 기록 (Thinking Record): thinking_record.json
  - 워크스페이스 (Workspace): 사고 작업 공간
  - 체크포인트 (Checkpoint): 사고 진행 저장점
```

**4. 에이전트 관련 (Ch 16)**:
```yaml
추가 용어:
  - 에이전트 (Agent): 사고 클러스터의 구현체
  - 도구 (Tool): 에이전트가 사용하는 기능
  - 프롬프트 (Prompt): AI에게 전달하는 인스트럭션
```

#### 용어집 구조 제안

```markdown
# 용어집 (Glossary)

## A
- Agent (에이전트): ...

## C
- Checkpoint (체크포인트): ...
- Core Values (핵심 가치): ...

## E
- Execution Item (실행 항목): ...
- Execution Worker (실행 워커): ...

## M
- Meta Coordinator (메타 조율자): ...
- Mission (미션): ...

## T
- Thinking Cluster (사고 클러스터): ...
- Thinking Coordinator (사고 조율자): ...
- Thinking Flow (사고 흐름): ...
- Thinking Process (사고 프로세스): ...
- Thinking Record (사고 기록): ...
- Thinking Stage (사고 단계): ...
- Thinking State (사고 상태): ...

## V
- Vision (비전): ...

## W
- Workspace (워크스페이스): ...
```

## 한영 병기 규칙

### 현재 규칙 (우수함)

**패턴 1: 첫 등장 시 영문 병기**
```markdown
사고 클러스터(Thinking Cluster)는...
```

**패턴 2: 이후 한글만 사용**
```markdown
이 사고 클러스터는...
```

**패턴 3: 강조 시 영문 병기 반복**
```markdown
**사고 클러스터(Thinking Cluster)**의 핵심은...
```

### 추가 권장 사항

**코드/YAML에서는 영문 사용**:
```yaml
thinking_cluster:  # 영문
  name: "콘텐츠 생성 사고 클러스터"  # 한글은 값에만
  coordinator:
    role: "사고 조율자"
```

**파일명은 영문**:
```
thinking_state.json  ✅
사고_상태.json  ❌
```
