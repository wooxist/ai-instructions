# Task 05: 예제 및 다이어그램 검토

**예상 소요 시간**: 4시간  
**우선순위**: High  
**담당**: AI Agent

---

## 목표

2권의 모든 코드 예제와 다이어그램의 품질을 확인하고, 명확성과 일관성을 검증합니다.

---

## 입력

### 검토 대상 문서
- `book/vol-2/part-*/ch-*/*.md` - 모든 2권 챕터
- 특히 Part 4 (Ch 11-16) - 실전 사례 중심

### 검토 기준
- 코드 예제 실행 가능성
- 다이어그램 명확성 및 정확성
- 예제 난이도 적절성
- 시각 자료 일관성

---

## 작업 절차

### 1단계: 코드 예제 검토

#### 검토 항목
- [ ] 문법 오류 확인
- [ ] 실행 가능성 검증
- [ ] 주석 충분성
- [ ] 변수명 명확성
- [ ] 에러 처리 포함 여부

#### 검토 방법
```markdown
각 코드 블록에 대해:
1. 언어 태그 확인 (```yaml, ```python 등)
2. 들여쓰기 일관성
3. 예제 목적 명확성
4. 출력 결과 제시 여부
```

### 2단계: 다이어그램 검토

#### 검토 항목
- [ ] Mermaid 문법 정확성
- [ ] 관계 표현 명확성
- [ ] 레이블 가독성
- [ ] 색상/스타일 일관성
- [ ] 크기 적절성

#### 검토 방법
```markdown
각 다이어그램에 대해:
1. 렌더링 가능 여부
2. 핵심 메시지 전달력
3. 설명과의 일치성
4. 복잡도 적절성
```

### 3단계: 예제 난이도 평가

#### 평가 기준
- **초급**: 기본 문법/구조만 사용
- **중급**: 여러 개념 조합
- **고급**: 복잡한 로직/최적화

#### 검토 항목
- [ ] 챕터 진행에 따른 난이도 상승
- [ ] 독자 수준 고려
- [ ] 단계별 설명 충분성

### 4단계: 시각 자료 일관성

#### 검토 항목
- [ ] 다이어그램 스타일 통일
- [ ] 색상 팔레트 일관성
- [ ] 폰트/크기 일관성
- [ ] 아이콘 사용 일관성
