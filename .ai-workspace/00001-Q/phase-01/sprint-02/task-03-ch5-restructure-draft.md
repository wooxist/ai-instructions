## 새로운 5.3.1절 내용

### 5.3.1 에이전트 분할이 필요한 신호 인식하기

"내 프롬프트가 너무 복잡해진 것 같은데, 에이전트를 나눠야 할까?" 이 질문에 답하는 가장 좋은 방법은 **결과물을 관찰하는 것**입니다. 일반 사용자는 토큰 수나 컨텍스트 윈도우를 직접 제어할 수 없지만, AI의 행동을 통해 분할 시점을 파악할 수 있습니다.

#### 분할이 필요한 4가지 신호

**신호 1: AI가 앞부분의 지시를 잊어버린다**
- 증상: 프롬프트 초반에 강조한 중요한 규칙을 무시하거나, "아까 말했던 대로"가 통하지 않는다.
- 원인: 지시가 너무 길어져 AI가 초반 내용을 기억하지 못함.
- 해결: 하나의 긴 프롬프트를 여러 개의 짧은 에이전트로 분할.

**예시:**
- **문제**: "고객 리뷰를 분석하되, 반드시 원문을 인용하고, 긍정/부정을 분류하고, 핵심 키워드를 추출하고, 월간 트렌드를 분석하고, 보고서를 작성해줘. 아, 그리고 처음에 말했듯이 반드시 원문 인용해야 해!"
  → AI는 마지막 요청만 기억하고 원문 인용을 빠뜨림.

- **해결**: 4개의 에이전트로 분할
  1. 분류 에이전트 (긍정/부정)
  2. 추출 에이전트 (키워드 + 원문 인용)
  3. 분석 에이전트 (트렌드)
  4. 보고서 작성 에이전트

**신호 2: 결과가 요청과 다르거나 일관성이 없다**
- 증상: 같은 프롬프트에 대해 매번 다른 형식이나 품질의 결과가 나온다.
- 원인: 프롬프트가 여러 가지 일을 동시에 요구해 AI가 혼란스러움.
- 해결: 각 작업을 전담하는 전문 에이전트로 분리.

**예시:**
- **문제**: "이 계약서를 법률 검토하고, 재무적 위험도 분석하고, 일반인이 이해하기 쉽게 요약해줘."
  → AI는 한 번은 법률 용어로, 한 번은 쉬운 말로, 한 번은 재무 중심으로 설명해서 혼란스러움.

- **해결**: 3개의 전문 에이전트
  1. 법률 검토 에이전트 (법률 전문가 페르소나)
  2. 재무 분석 에이전트 (재무 전문가 페르소나)
  3. 요약 에이전트 (일반인 눈높이)

**신호 3: 특정 부분을 수정하려면 전체를 다시 해야 한다**
- 증상: 작은 부분의 오류를 고치려고 하면 전체 프롬프트를 다시 실행해야 함.
- 원인: 모든 것이 하나의 프롬프트에 얽혀 있어 부분 수정이 불가능.
- 해결: 독립적으로 수정 가능한 모듈형 에이전트 설계.

**예시:**
- **문제**: "기사를 읽고 요약하고 번역하고 SNS 게시물로 만들어줘."
  → 번역 부분만 수정하고 싶은데 전체를 다시 실행해야 함.

- **해결**: 파이프라인형 에이전트
  1. 요약 에이전트 → 2. 번역 에이전트 → 3. SNS 작성 에이전트
  → 번역만 수정하고 싶으면 2번부터 재실행.

**신호 4: 같은 작업을 반복적으로 요청하게 된다**
- 증상: 비슷한 프롬프트를 조금씩 바꿔가며 여러 번 사용.
- 원인: 재사용 가능한 에이전트가 아닌 일회성 프롬프트 사용.
- 해결: 재사용 가능한 범용 에이전트 설계.

**예시:**
- **문제**: 매일 다른 데이터에 대해 "이 엑셀 파일을 읽고 차트를 만들고 인사이트를 뽑아줘"를 반복.
  
- **해결**: 1개의 재사용 가능 에이전트
  - "데이터 분석 에이전트": 어떤 엑셀이든 받아서 차트 + 인사이트 생성
  - 매일 다른 파일을 입력으로 제공하기만 하면 됨.

#### 실용적 판단 기준

다음 질문에 2개 이상 "예"라면, 에이전트 분할을 고려하세요:

1. ☐ 프롬프트를 설명하는 데 3분 이상 걸리나요?
2. ☐ AI가 요청한 내용 중 일부를 자주 빠뜨리나요?
3. ☐ 같은 프롬프트로 매번 다른 품질의 결과가 나오나요?
4. ☐ 작은 수정을 위해 전체를 다시 실행해야 하나요?
5. ☐ 이 프롬프트를 다른 상황에서도 재사용하고 싶으신가요?

#### 점진적 분할 접근법

처음부터 완벽하게 에이전트를 나눌 필요는 없습니다. 다음 단계를 따라가세요:

**1단계: 하나로 시작**
- 먼저 하나의 프롬프트로 작업을 시도해보세요.
- 결과물의 품질과 일관성을 확인하세요.

**2단계: 문제 신호 감지**
- 위의 4가지 신호 중 하나라도 보이나요?
- 어떤 부분에서 문제가 발생하나요?

**3단계: 문제 부분만 분리**
- 전체를 한꺼번에 나누지 말고, 가장 문제가 되는 부분만 먼저 분리하세요.
- 예: "요약 + 번역 + SNS 작성"에서 번역이 문제라면, 번역만 별도 에이전트로.

**4단계: 테스트 및 반복**
- 분리한 에이전트로 다시 시도해보세요.
- 여전히 문제가 있다면 추가로 분리하세요.
