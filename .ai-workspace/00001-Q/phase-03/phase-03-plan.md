# Phase 03: 인간 검토 및 최종 개선

**시작**: 2025-10-15  
**담당**: 인간(검토) + AI Agent(개선 지원)  
**방식**: 1 서브챕터 단위 검토

---

## 검토 방식

### 작업 단위
**1 서브챕터 = 1 검토 단위**

예시:
- ✅ 11.1.1 단일 목표에서 사고 단위로
- ✅ 11.1.2 인간과 AI의 역할
- ✅ 11.1.3 사고 vs 실행 항목

### 검토 프로세스

```
인간이 서브챕터 검토
    ↓
개선 사항 발견 시
    ↓
개선 요청 (이슈 설명)
    ↓
AI Agent: 개선 계획 수립
    ↓
인간 승인
    ↓
AI Agent: 개선 실행
    ↓
인간 확인
    ↓
다음 서브챕터
```

---

## AI Agent 역할

### 1. 개선 계획 수립
- 문제 분석
- 해결 방안 제시
- 영향 범위 파악
- 예상 소요 시간

### 2. 개선 실행
- 승인된 계획 실행
- 관련 파일 수정
- 일관성 유지
- 변경 사항 문서화

### 3. 진행 상황 추적
- 검토 완료 서브챕터 기록
- 개선 이력 관리
- 다음 검토 대상 안내

---

## 개선 요청 방법

검토 시 자유롭게 말씀해주세요:

**간단한 요청**:
- "이 부분 표현이 어색해요"
- "예제가 더 필요해요"
- "다이어그램을 추가해주세요"

**상세한 요청**:
- "11.1.2에서 역할 구분이 명확하지 않아요. 표를 추가해주세요"
- "11.2.1 예제가 너무 복잡해요. 더 간단한 예제로 바꿔주세요"

AI가 자동으로:
1. 문제 분석
2. 개선 계획 제시
3. 승인 후 실행
4. 결과 보고

---

## 검토 시작 방법

**검토 시작하려면**:
1. 검토할 서브챕터 말씀해주세요
2. 예: "11.1.1부터 시작할게요"
3. 예: "Preface부터 검토할게요"

**개선 요청 시**:
1. 발견한 문제 설명
2. AI가 개선 계획 제시
3. 승인 또는 수정 요청
4. AI가 실행
