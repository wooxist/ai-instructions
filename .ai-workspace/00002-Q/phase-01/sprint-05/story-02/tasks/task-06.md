# Task 6: 전체 검토 및 통합

## 목적
수정된 모든 섹션을 검토하고, 섹션 간 연결성, 일관성, 전체적인 독자 경험을 확인하여 Chapter 3을 최종 완성합니다.

## 배경
Task 1-5에서 개별 섹션들을 수정했지만, 각 수정이 독립적으로 이루어졌기 때문에 전체적인 흐름과 일관성을 점검할 필요가 있습니다.

## 입력
- Task 1-5에서 수정된 Chapter 3 전체

## 작업 내용

### 1. 전체 흐름 검토

#### 섹션 간 연결성 확인
- [ ] 3.1 → 3.2: 5가지 속성에서 9가지 구성 요소로의 전환이 자연스러운가?
- [ ] 3.2 → 3.3: 구성 요소 설명에서 템플릿으로의 전환이 매끄러운가?
- [ ] 3.3 → 3.4: 템플릿에서 실전 예시로의 연결이 명확한가?
- [ ] 3.4 → 3.5: 예시에서 체크리스트로의 전환이 논리적인가?

#### 중복 내용 확인
- [ ] 같은 내용이 여러 섹션에서 반복되지 않는가?
- [ ] 특히 "필수 5가지"에 대한 설명이 3.2와 다른 곳에서 중복되지 않는가?

#### 누락 내용 확인
- [ ] 새로 추가된 개념(레이어 구조, 명확성 스펙트럼)이 요약에 반영되었는가?
- [ ] 실습 과제가 새로운 내용을 반영하는가?

### 2. 일관성 점검

#### 용어 일관성
- [ ] "시스템 속성" 용어가 일관되게 사용되는가?
- [ ] "필수/권장/선택" 표현이 일관되는가?
- [ ] 이모지(⭐🔵⚪) 사용이 일관되는가?

#### 문체 일관성
- [ ] 독자 친화적 어투("~예요", "~죠")가 일관되게 유지되는가?
- [ ] 존댓말/반말 혼용이 없는가?
- [ ] 새로 추가된 부분의 톤이 기존과 어울리는가?

#### 형식 일관성
- [ ] 마크다운 헤딩 레벨이 일관되는가?
- [ ] 코드 블록 형식이 통일되어 있는가?
- [ ] 표 형식이 일관되는가?

### 3. 독자 경험 점검

#### 학습 곡선
- [ ] 개념이 점진적으로 도입되는가?
- [ ] 어려운 내용 앞에 충분한 설명이 있는가?
- [ ] 예시가 개념을 이해하는 데 도움이 되는가?

#### 실용성
- [ ] 독자가 바로 적용할 수 있는 도구가 충분한가?
- [ ] 템플릿/체크리스트가 실제로 사용 가능한가?
- [ ] 다양한 도메인 예시가 전이 학습을 돕는가?

#### 분량 적정성
- [ ] 전체 분량이 과도하지 않은가? (원본 대비 +4500자 이내)
- [ ] 각 섹션의 길이가 적절한가?
- [ ] 장황한 설명이나 불필요한 반복이 없는가?

### 4. 품질 확인

#### 정확성
- [ ] 기술적 내용에 오류가 없는가?
- [ ] 예시가 정확하고 현실적인가?
- [ ] 점수/수치가 합리적인가?

#### 완성도
- [ ] 모든 섹션이 완전히 작성되었는가?
- [ ] TODO나 placeholder가 남아있지 않은가?
- [ ] 참고 자료가 적절한가?

### 5. 최종 점검

#### Story 완료 기준 재확인
- [ ] Task 1-3 (우선순위 높음) 완료됨
- [ ] 추가된 내용이 독자 친화적 문체 유지
- [ ] 기존 내용과 자연스럽게 통합됨
- [ ] 전체 분량이 과도하게 증가하지 않음 (4500자 이내)

#### 실습 과제 업데이트
- [ ] 실습 과제가 새로운 내용을 반영하는가?
- [ ] 특히 "과제 2: 9가지 구성 요소 템플릿 작성"이 새로운 우선순위 설명을 반영하는가?

## 출력
최종 검토 완료된 Chapter 3 파일

## 완료 기준
- [ ] 전체 흐름이 자연스러움
- [ ] 중복/누락 없음
- [ ] 용어/문체/형식 일관성 확보
- [ ] 독자 경험 최적화
- [ ] 품질 기준 충족
- [ ] Story 완료 기준 모두 만족

## 실행 방법
1. Chapter 3 파일 전체를 읽는다
2. 위의 체크리스트를 하나씩 점검한다
3. 문제가 발견되면 즉시 수정한다
4. 모든 항목이 만족될 때까지 반복한다
5. 최종 승인을 요청한다
