# Task 5: 점수 기반 체크리스트 전환

## 목적
3.5절 전체를 5점 만점 체크리스트로 재작성하여 독자가 인스트럭션 품질을 정량적으로 평가하고 개선 우선순위를 판단할 수 있게 합니다.

## 배경
현재 3.5절의 체크리스트는 Yes/No 이진 판단 방식으로, "부분적으로 만족" 케이스를 다루지 못합니다. 예를 들어 "모호한 표현이 없는가?"는 1개만 있어도 No가 되어 너무 엄격합니다. 점수 기반으로 바꾸면 현재 상태를 정확히 진단하고 개선 우선순위를 설정할 수 있습니다.

## 입력
- `/doc/book1/vol-1-part-1-chapter-03.md` (현재 버전)
- 개선안: 이전 대화의 "개선 제안 #6" 내용

## 작업 내용

### 1. 교체할 섹션 확인
3.5절 "시스템 체크리스트" 전체를 찾습니다. 현재는 5가지 속성에 대한 Yes/No 체크리스트입니다.

### 2. 교체할 내용
3.5절 전체를 다음 구조로 전면 재작성합니다:

**새로운 구조**:
1. 평가 방법 설명 (5점 척도)
2. 전체 점수 해석 기준
3. 5가지 속성별 평가 항목 (각 5점 만점)
   - 재사용 가능성
   - 일관성 보장
   - 개선 가능성
   - 협업 가능성
   - 확장 가능성
4. 종합 평가 시트
5. 실전 적용 예시

**각 속성의 구조**:
```markdown
### N. [속성명] (5점 만점)

#### 평가 항목
- [ ] **항목1 (2점)**: 설명
- [ ] **항목2 (2점)**: 설명
- [ ] **항목3 (1점)**: 설명

**점수**: __/5점

**개선 우선순위**:
- 0-2점: 즉시 개선 필요
- 3-4점: 점진적 개선
- 5점: 유지

**개선 팁**:
- 구체적 개선 방법 1
- 구체적 개선 방법 2
```

### 3. 상세 내용
개선안 문서의 "개선 제안 #6"에 있는 전체 내용을 그대로 사용합니다. 약 1200자 분량입니다.

주요 포함 내용:
- 평가 방법 (5점 척도 설명)
- 전체 점수 해석 (20-25점 우수, 15-19점 실용적, 10-14점 최소, 0-9점 불충분)
- 5가지 속성별 세부 평가 항목과 점수 배분
- 개선 우선순위 가이드라인
- 개선 팁
- 종합 평가 시트
- "주간 판매 보고서 v1.0" 실전 평가 예시

### 4. 검증 사항
- [ ] 5점 척도가 명확한가
- [ ] 점수 배분이 합리적인가
- [ ] 개선 우선순위 기준이 실용적인가
- [ ] 독자 친화적 어투를 유지하는가
- [ ] 전체 내용이 약 1200자인가

## 출력
수정된 3.5절이 포함된 Chapter 3 파일

## 완료 기준
- [ ] 3.5절이 전면 재작성됨
- [ ] 5점 척도 평가 시스템 도입됨
- [ ] 5가지 속성 모두 점수화됨
- [ ] 실전 예시가 포함됨
- [ ] 독자 친화적 문체 유지
- [ ] 분량이 약 1200자 추가됨

1. Chapter 3 파일을 연다
2. 3.5절 "시스템 체크리스트" 전체를 찾는다
3. Filesystem:edit_file을 사용하여 섹션 전체를 교체한다
4. 수정된 파일을 확인한다
