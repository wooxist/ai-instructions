# Task 4: 명확성 스펙트럼 추가 (선택적)

## 목적
3.2절의 "충분히 명확한 인스트럭션이란?" 섹션에 Level 0-3의 명확성 단계를 추가하여 독자가 자신의 인스트럭션 품질을 판단할 수 있게 합니다.

## 배경
현재 3.2절은 "충분히 명확한"의 의미를 "추가 대화 없이 실행 가능"으로만 설명하지만, 이는 이진적 판단입니다. 실제로는 명확성에 단계가 있으며, 독자는 자신이 어느 단계에 있는지, 어디를 목표로 해야 하는지 알아야 합니다.

## 입력
- `/doc/book1/vol-1-part-1-chapter-03.md` (현재 버전)
- 개선안: 이전 대화의 "개선 제안 #3" 내용

## 작업 내용

### 1. 삽입 위치 확인
3.2절의 "충분히 명확한 인스트럭션이란?" 섹션 마지막에 새로운 하위 섹션을 추가합니다.

### 2. 추가할 내용
다음 내용을 추가합니다:

```markdown
### 명확성의 3단계 스펙트럼

"충분히 명확하다"는 것은 상대적입니다. 여러분의 인스트럭션이 현재 어느 
수준인지 확인해보세요:

#### Level 0: 채팅 의존 (시스템 아님)
```
"보고서 만들어줘"
```
→ AI 반응: "어떤 형식으로 하시겠습니까?" (3회 이상 질문)  
→ 문제: 매번 대화로 빈칸 채우기, 재사용 불가능

#### Level 1: 최소 실행 가능 (시스템의 시작)
```markdown
목적, 역할, 입력, 처리, 출력 명시 (필수 5가지)
```
→ AI 반응: "확인했습니다" (바로 실행 가능)  
→ 특징: 실행은 되지만 결과가 조금씩 다를 수 있음  
→ 적용: v1.0 단계

#### Level 2: 안정적 품질 (실전 사용)
```markdown
Level 1 + 제약 조건 + 평가 기준 (필수 5 + 권장 2)
```
→ AI 반응: 조용히 실행하고 체크리스트로 자가 검증  
→ 특징: 일관된 품질, 실패율 10% 미만  
→ 적용: v2.0 단계, 팀 공유 시작

#### Level 3: 완전 명확 (성숙한 시스템)
```markdown
Level 2 + 기초 지식 + 출력 템플릿 (9가지 전체)
```
→ AI 반응: 완벽히 이해하고 예시와 동일한 품질로 출력  
→ 특징: 신입도 설명 없이 사용 가능, 실패율 1% 미만  
→ 적용: v3.0+, 조직 표준화

**판단 기준**:
- Level 1 미만: 아직 시스템이 아님 → 필수 5가지부터 채우기
- Level 1-2: 개인 사용은 OK → 팀 공유 전 Level 2로 올리기
- Level 2-3: 팀 사용 안정 → 필요시 Level 3으로 성숙화

대부분의 경우 **Level 2면 충분**합니다. Level 3은 조직 전체 표준이나 
장기 운영 시스템에만 필요합니다.
```

### 3. 검증 사항
- [ ] 4단계가 명확히 구분되는가
- [ ] 각 단계의 특징이 구체적인가
- [ ] 독자가 자신의 수준을 판단할 수 있는가
- [ ] 독자 친화적 어투를 유지하는가
- [ ] 추가 내용이 약 500자인가

## 출력
수정된 3.2절이 포함된 Chapter 3 파일

## 완료 기준
- [ ] 명확성 스펙트럼이 추가됨
- [ ] 4단계가 명확히 구분됨
- [ ] 판단 기준이 실용적임
- [ ] 독자 친화적 문체 유지
- [ ] 분량이 약 500자 추가됨

## 실행 방법
1. Chapter 3 파일을 연다
2. "충분히 명확한 인스트럭션이란?" 섹션 마지막을 찾는다
3. Filesystem:edit_file을 사용하여 내용을 추가한다
4. 수정된 파일을 확인한다
