# Task 1: 5가지 속성 계층 구조화

## 목적
3.1절에 시스템 속성의 3단계 레이어(기반/성장/확장) 설명을 추가하여 5가지 속성이 독립적이 아닌 계층적 관계임을 명확히 합니다.

## 배경
현재 3.1절은 5가지 속성을 나열식으로 설명하여, 독자가 속성 간 관계와 우선순위를 파악하기 어렵습니다. 특히 "재사용 가능"과 "확장 가능"의 차이가 불명확합니다.

## 입력
- `/doc/book1/vol-1-part-1-chapter-03.md` (현재 버전)
- 개선안: 이전 대화의 "개선 제안 #1" 내용

## 작업 내용

### 1. 삽입 위치 확인
3.1절의 "좋은 시스템은 다섯 가지 핵심 속성을 가지고 있습니다. 이 속성들을 하나씩 살펴보면서..." 문장 다음에 새로운 단락을 삽입합니다.

### 2. 추가할 내용
다음 내용을 추가합니다:

```markdown
이 속성들은 독립적이지 않고, 세 가지 레이어로 계층을 이룹니다:

**🏗️ 기반 레이어**: 시스템의 최소 조건
1. **재사용 가능** - 반복 사용의 기반
2. **일관성 보장** - 신뢰할 수 있는 결과

이 두 가지가 없으면 시스템이라고 부를 수 없습니다. 한 번만 쓰고 버리거나, 
매번 다른 결과가 나온다면 의미가 없으니까요.

**📈 성장 레이어**: 시스템의 진화
3. **개선 가능** - 시간에 따라 발전
4. **협업 가능** - 공간에 따라 확산

기반이 탄탄하면 이제 시스템을 더 나아지게 만들 수 있습니다. 혼자서 점점 
개선하고, 팀 전체로 확산되는 거죠.

**🚀 확장 레이어**: 시스템의 스케일
5. **확장 가능** - 규모와 범위로 성장

성장이 안정화되면 이제 더 큰 규모, 더 넓은 범위로 확장할 수 있습니다.

이 계층 구조를 염두에 두고 각 속성을 하나씩 살펴보겠습니다.
```

### 3. 검증 사항
- [ ] 기존 문장과 자연스럽게 연결되는가
- [ ] 독자 친화적 어투를 유지하는가 ("~예요", "~죠")
- [ ] 이모지 사용이 과하지 않은가
- [ ] 추가 내용이 약 400자인가

## 출력
수정된 3.1절이 포함된 Chapter 3 파일

## 완료 기준
- [ ] 계층 구조 설명이 삽입됨
- [ ] 기존 내용과 자연스럽게 통합됨
- [ ] 독자 친화적 문체 유지
- [ ] 분량이 400자 이내로 추가됨

## 실행 방법
1. Chapter 3 파일을 연다
2. 3.1절의 삽입 위치를 찾는다
3. Filesystem:edit_file을 사용하여 내용을 추가한다
4. 수정된 파일을 확인한다
