# Task 4: 기존 파일 정리 및 검증

## 목적
book1에서 기존 8장, 9장, 10장 파일을 삭제하고, 전체 목차와 파일의 일관성을 검증한다.

## 입력
- Task 1, 2, 3의 완료 상태
- `doc/book1/` 디렉토리의 모든 파일

## 출력
- 정리된 book1 디렉토리
- 검증 보고서 (구두 보고 가능)

## 실행 방법

### 1단계: 파일 삭제 확인

#### 1-1. Task 1, 2, 3 완료 여부 확인
- [ ] book3-draft에 8장, 9장이 정상 복사되었는가?
- [ ] 새로운 vol-1-part-3-chapter-08.md가 생성되었는가?
- [ ] vol-1-index.md가 수정되었는가?

모두 확인되면 다음 단계 진행

#### 1-2. 기존 파일 삭제 목록
다음 파일들을 book1에서 삭제:
1. `doc/book1/vol-1-part-3-chapter-08.md` (기존 8장 - 성능 최적화)
2. `doc/book1/vol-1-part-3-chapter-09.md` (기존 9장 - 평가와 검증)
3. `doc/book1/vol-1-part-4-chapter-10.md` (기존 10장 - 실전 패턴)

**주의**: 삭제 전 한 번 더 확인!
- book3-draft에 8장, 9장이 있는가?
- vol-1-part-3-chapter-08.md (새 8장)가 생성되었는가?

### 2단계: 파일 목록 검증

#### 2-1. book1 디렉토리 최종 상태 확인
`doc/book1/` 디렉토리에 다음 파일들만 존재해야 함:
- [ ] vol-1-index.md
- [ ] vol-1-part-0-preface.md
- [ ] vol-1-part-1-chapter-01.md
- [ ] vol-1-part-1-chapter-02.md
- [ ] vol-1-part-1-chapter-03.md
- [ ] vol-1-part-1-complete.md (있는 경우)
- [ ] vol-1-part-2-chapter-04.md
- [ ] vol-1-part-2-chapter-05.md
- [ ] vol-1-part-2-chapter-06.md
- [ ] vol-1-part-3-chapter-07.md
- [ ] vol-1-part-3-chapter-08.md (새로운 8장 - 실전 패턴)

#### 2-2. book3-draft 디렉토리 확인
`doc/book3-draft/` 디렉토리에 다음 파일들이 존재해야 함:
- [ ] README.md
- [ ] vol-1-part-3-chapter-08.md (이전 8장 - 성능 최적화)
- [ ] vol-1-part-3-chapter-09.md (이전 9장 - 평가와 검증)

### 3단계: 내용 일관성 검증

#### 3-1. vol-1-index.md 검증
다음 항목들 확인:
- [ ] Part 3에 7장, 8장만 존재하는가?
- [ ] Part 4가 삭제되었는가?
- [ ] 모든 링크가 올바른 파일을 가리키는가?
- [ ] 8장 링크가 vol-1-part-3-chapter-08.md를 가리키는가?
- [ ] 섹션 번호(8.1, 8.2, 8.3)가 올바른가?

#### 3-2. 새로운 8장 파일 검증
`doc/book1/vol-1-part-3-chapter-08.md` 확인:
- [ ] H1 제목이 "# 8장. 실전 패턴: 단일 에이전트 설계"인가?
- [ ] 모든 섹션 번호가 8.X 형태인가? (10.X가 남아있지 않은가?)
- [ ] 본문에 "10장" 텍스트가 남아있지 않은가?
- [ ] 앵커 링크가 #8X 형태인가? (#10X가 남아있지 않은가?)

#### 3-3. 다른 파일에서 참조 확인
다음 파일들을 검색하여 삭제된 8장, 9장을 참조하는 부분이 있는지 확인:
- [ ] vol-1-part-0-preface.md
- [ ] vol-1-part-1-chapter-01.md ~ 03.md
- [ ] vol-1-part-2-chapter-04.md ~ 06.md
- [ ] vol-1-part-3-chapter-07.md
- [ ] vol-1-part-3-chapter-08.md (새 8장)

**확인 방법**: "8장", "9장", "성능 최적화", "평가와 검증" 등의 키워드로 검색

### 4단계: 최종 구조 확인

전체 Book1 구조가 다음과 같은지 확인:
```
Part 0: 서문
Part 1: 프롬프트와 인스트럭션의 기초 (1-3장)
Part 2: 설계 원칙과 구성 요소 (4-6장)
Part 3: 워크플로우 패턴과 실전 (7-8장)
  - 7장: 기본 워크플로우 패턴과 처리
  - 8장: 실전 패턴 - 단일 에이전트 설계
```

## 완료 기준
- [ ] book1에서 기존 8장, 9장, 10장 파일이 삭제됨
- [ ] book1 디렉토리에 올바른 파일들만 존재함
- [ ] book3-draft에 이전 8장, 9장이 보관됨
- [ ] vol-1-index.md의 모든 링크가 작동함
- [ ] 새로운 8장 파일의 모든 번호가 올바름
- [ ] 다른 파일에서 삭제된 8장, 9장을 참조하지 않음
- [ ] 전체 구조가 일관성 있음

## 주의사항
- 파일 삭제는 신중하게! 반드시 백업(book3-draft) 확인 후 진행
- 검증은 꼼꼼히! 링크가 깨지면 독자가 불편함
- 의심스러운 부분은 사람(Daewook)에게 확인 요청
