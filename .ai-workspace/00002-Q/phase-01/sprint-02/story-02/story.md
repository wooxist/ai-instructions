# Story 2: 8장, 9장 삭제 및 Book1 목차 재구성

## 목표
8장(성능 최적화)과 9장(평가와 검증)을 Book1에서 제거하고 Book3-draft로 이동하며, 10장을 8장으로 재넘버링하여 Book1 구조를 일반 사용자 중심으로 정리한다.

## 배경
8장과 9장은 정량적 측정 도구(API, 토큰 수 측정)를 사용할 수 있는 프로그래밍 가능한 독자를 대상으로 한 내용이다. Book1의 타겟 독자인 일반 사용자(코딩 경험 없거나 기초적)에게는 실용성이 낮으며, 측정 도구 없이는 "여러 번 실행해보고 비교하세요" 수준의 조언만 가능하다.

따라서 이 두 장을 향후 3권(고급 최적화와 측정)으로 분리하고, Book1은 실제로 적용 가능한 내용(1-7장, 10장)으로만 구성한다.

## 기대 효과
- Book1의 타겟 독자(일반 사용자)와 내용의 일치도가 높아진다.
- 독자가 실제로 활용할 수 없는 내용이 제거되어 실용성이 향상된다.
- Book1이 더 간결하고 완결성 있는 구조가 된다.
- 향후 3권 작업 시 8장, 9장을 재활용할 수 있다.

## Workflow

### Task 1: 8장, 9장을 Book3-draft로 이동
→ `tasks/task-01.md`
- 8장, 9장 파일을 book3-draft로 복사
- book3-draft/README.md 작성 (보관 목적 설명)
- 원본 파일은 유지 (다음 Task에서 확인 후 처리)

### Task 2: 10장 파일명 변경 (10장 → 8장)
→ `tasks/task-02.md`
- vol-1-part-4-chapter-10.md → vol-1-part-3-chapter-08.md
- 파일 내부의 모든 "10장" 텍스트를 "8장"으로 변경
- Part 4 → Part 3으로 변경

### Task 3: vol-1-index.md 목차 수정
→ `tasks/task-03.md`
- Part 3 제목 변경: "워크플로우, 성능과 평가" → "워크플로우 패턴"
- 8장, 9장 항목 삭제
- 10장을 8장으로 수정하고 Part 3으로 이동
- "1권에서 배운 것" 섹션 업데이트
- "이제 할 수 있는 것" 섹션 업데이트

### Task 4: 기존 8,9,10장 파일 정리 및 검증
→ `tasks/task-04.md`
- book1에서 기존 8장, 9장, 10장 파일 삭제
- 전체 목차 일관성 검증
- 다른 섹션에서 8,9장 참조 여부 확인
