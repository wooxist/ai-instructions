# Workflow: 챕터 검토 프로세스

**용도**: 모든 챕터 검토 작업에 공통 적용

---

## 작업 흐름

### Task 01: 인간 검토
**목표**: 독자 시점에서 챕터 검토

**실행 방법**: 
대상 챕터를 독자 입장에서 읽고 다음 관점으로 평가
- 내용 이해도: 설명이 명확한가?
- 표현 적절성: 용어와 문체가 적절한가?
- 예제 품질: 예제가 이해를 돕는가?
- 흐름 자연스러움: 문단 간 연결이 매끄러운가?

**산출물**: 
- 개선 필요: 피드백 (문제점 + 개선 제안)
- 개선 불필요: "검토 완료, 개선 불필요" 확인

---

### Task 02: AI 개선 계획 수립
**조건**: Task 01에서 개선이 필요하다고 판단된 경우만 실행

**목표**: 피드백 기반 개선 계획 수립

**실행 방법**:
- 문제점 분석
- 해결 방안 제시
- 영향 범위 파악
- 수정 전후 비교

**산출물**: 개선 계획 문서

---

### Task 03: 인간 승인
**목표**: 개선 계획 검토 및 승인

**선택지**:
- ✅ 승인: Task 04로 진행
- 🔄 수정 요청: Task 02로 복귀
- ❌ 반려: 처음부터 재검토

---

### Task 04: AI 개선 실행
**조건**: Task 03 승인 완료

**목표**: 승인된 계획대로 파일 수정

**실행 방법**: 
- 계획된 수정 사항 적용
- 파일 저장

**산출물**: 수정된 챕터 파일

---

### Task 05: 인간 최종 확인
**목표**: 수정 결과 확인

**선택지**:
- ✅ 확인 완료: Story 완료
- 🔄 추가 수정: Task 02로 복귀

---

## 완료 기준

- [ ] 인간 검토 완료
- [ ] 개선 사항 반영 완료 (필요 시)
- [ ] 최종 확인 완료
- [ ] 파일 저장 완료

---

## 예외 처리

**Task 01에서 개선 불필요 판단 시**:
→ 바로 완료 처리 (Task 02-05 생략)

**Task 03에서 반려 시**:
→ Task 02로 돌아가 계획 재수립

**Task 05에서 추가 수정 요청 시**:
→ Task 02로 돌아가 수정 계획 수립
