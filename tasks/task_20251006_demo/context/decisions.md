# 아키텍처 결정 기록 (ADR)

## ADR-001: React 선택
**날짜**: 2025-10-06  
**상태**: 승인됨

### 컨텍스트
간단한 할일 앱을 만들어야 하며, 기술 스택을 선택해야 함.

### 결정
React를 사용하기로 결정

### 근거
1. **컴포넌트 재사용성**: 향후 확장 시 유리
2. **상태 관리 간편**: useState 훅으로 간단한 상태 관리
3. **생태계**: 풍부한 라이브러리와 커뮤니티
4. **학습 효과**: 사용자가 React 학습 가능

### 대안
- Vanilla JS: 가장 간단하지만 확장성 낮음
- Vue: 좋은 선택이나 React가 더 보편적

### 결과
React artifact로 구현 예정

## ADR-002: LocalStorage 사용
**날짜**: 2025-10-06  
**상태**: 승인됨

### 컨텍스트
데이터 지속성을 위한 저장소 선택

### 결정
LocalStorage 사용

### 근거
1. **간단성**: 별도 백엔드 불필요
2. **즉시 사용 가능**: 모든 모던 브라우저 지원
3. **충분한 용량**: 할일 앱 용도로는 5MB 충분

### 대안
- IndexedDB: 과도하게 복잡
- 백엔드 API: 이 프로젝트에는 불필요

### 제약사항
- 단일 사용자만 지원
- 데이터는 브라우저에만 저장됨
- 브라우저 간 동기화 불가

## ADR-003: TailwindCSS 사용
**날짜**: 2025-10-06  
**상태**: 승인됨

### 컨텍스트
스타일링 방법 선택

### 결정
TailwindCSS 유틸리티 클래스 사용

### 근거
1. **빠른 개발**: 미리 정의된 클래스 활용
2. **일관성**: 디자인 시스템 자동 적용
3. **반응형**: 쉬운 반응형 디자인

### 대안
- 순수 CSS: 더 많은 코드 작성 필요
- CSS-in-JS: 이 프로젝트에는 과함

### 주의사항
- Artifact에서는 core utility classes만 사용 가능
- 커스텀 클래스나 설정 파일 불가
