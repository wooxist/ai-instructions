## 🔄 작업 프로세스

### 새 작업 시작 시
1. 백로그에서 작업 선택
2. `context/{task-name}.md` 생성 (작업 컨텍스트 기록)
3. 진행 상태를 "이번 주 우선순위" 또는 "다음 주 작업"으로 이동
4. 작업 진행

### 작업 완료 시
1. 산출물을 지정된 경로에 저장
2. "완료된 작업" 섹션에 기록 (날짜, 작업명, 산출물)
3. ROADMAP.md의 해당 항목 체크
4. `changelog.md`에 변경 내역 기록

### 작업 차단 시
1. 차단 사유를 `context/{task-name}.md`에 기록
2. 백로그로 다시 이동, 우선순위 재조정
3. 대체 작업 선택
