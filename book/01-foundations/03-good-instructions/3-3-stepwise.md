# 3.3 단계성 (Step-wise)

## 목적
복잡한 작업을 검증 가능한 단계로 분해한다.

## 요약
단계/순서/중간 산출물을 명시해 추적 가능하게 한다.

## 핵심 개념
- 단계 정의와 의존성
- 중간 출력/검증 포인트
- 실패 시 회복 절차

## 예시
- 계획 수립 → 초안 → 검토 → 개선 → 배포

## 실습
- 현재 과제를 4–6단계로 분해하고 검증 포인트 지정

## 체크리스트
- 각 단계의 완료 기준이 있는가?
- 중간 산출물 예시가 있는가?
- 역추적/롤백 방법이 있는가?
