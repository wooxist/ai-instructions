# 11장. 기본 사고 클러스터 (Basic Thinking Cluster)

**핵심 질문**: 하나의 목표를 여러 사고 단위로 어떻게 분해할까?

---

## 도입

이전 장들에서 우리는 AI와의 협업을 **명확한 인스트럭션**을 통해 수행하는 방법을 배웠습니다. 그러나 실제 업무 환경에서는 단순히 "이 작업을 해줘"라고 말하는 것만으로는 부족합니다. 복잡한 목표를 달성하려면 **사고를 체계적으로 설계**해야 합니다.

11장에서는 **기본 사고 클러스터(Basic Thinking Cluster)** 개념을 소개합니다. 이는 하나의 목표를 여러 사고 단위로 분해하고, 각 단위가 어떻게 협력하여 최종 결과를 만들어내는지를 다룹니다.

### 이 장에서 다룰 내용

1. **11.1 사고 클러스터 개념**: 사고 단위란 무엇이며, 왜 필요한가?
2. **11.2 기본 설계 패턴**: 3가지 핵심 패턴 (단순 분기, 순차 처리, 병렬 처리)
3. **11.3 실전 사례**: 콘텐츠 생성과 데이터 분석 사례를 통한 6계층 모델 적용

### AI 사고 생태계와 11장의 위치

이 책에서 소개한 **AI 사고 생태계 6계층 모델**(`report_kr.md`)을 기억하시나요?

```
1층: 미션 (Mission) - 조직의 존재 이유
2층: 핵심 가치 (Core Values) - 의사결정 기준 ⭐
3층: 비전 (Vision) - 미래 상태
4층: 목표 (Goal) - 전략적 목표
5층: 사고 클러스터 (Thinking Cluster) - 인간+AI 사고 ⭐ ← 11장 초점
6층: 실행 항목 (Execution Items) - 구체적 작업
```

**11장의 초점은 바로 5층: 사고 클러스터입니다.** 여기서는:

- 추상적인 목표(4층)를 구체적인 사고 프로세스로 어떻게 변환하는가?
- 사고 프로세스는 어떻게 실행 항목(6층)을 도출하는가?
- 핵심 가치(2층)가 사고 클러스터 내에서 어떻게 의사결정 기준이 되는가?

### 11장 vs 12장: 차이점

| 구분 | 11장 (기본 사고 클러스터) | 12장 (계층적 사고 클러스터) |
|------|--------------------------|--------------------------|
| **목표 수** | 단일 목표 | 복합 목표 (여러 개) |
| **사고 클러스터** | 하나의 클러스터 | 여러 클러스터 (계층적) |
| **조율자** | 단순 조율 | 메타 조율자 (상위 조율) |
| **복잡도** | 기본 (Basic) | 고급 (Advanced) |
| **예시** | 소셜 미디어 포스트 작성 | 전체 마케팅 캠페인 설계 |

**11장은 기초**입니다. 하나의 목표에 집중하여, 사고 클러스터가 어떻게 작동하는지 이해하는 것이 목표입니다.

### 학습 목표

이 장을 마치면 다음을 할 수 있습니다:

1. **사고 단위의 개념 이해**: 사고와 실행을 구분하고, 사고 클러스터가 왜 필요한지 설명할 수 있습니다.
2. **기본 패턴 적용**: 단순 분기, 순차 처리, 병렬 처리 패턴을 실제 문제에 적용할 수 있습니다.
3. **6계층 통합**: AI 사고 생태계의 6계층 모델을 사고 클러스터 설계에 통합할 수 있습니다.
4. **실전 설계**: 실제 업무 상황에서 사고 클러스터를 설계하고 구현할 수 있습니다.
