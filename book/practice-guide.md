# 실습 과제 모음 (Ch.01–14)

각 장별로 바로 실천할 수 있는 심화 실습 과제를 제공합니다. 각 과제는 목표, 단계, 예상 결과물, 자가 평가 기준, 난이도로 구성되어 있습니다.

## 01장 프롬프트와 인스트럭션 이해하기 — 실습 과제
- 난이도: [기본]
- 목표: 나쁜 프롬프트를 좋은 프롬프트로 개선한다.
- 단계별 가이드:
  1) 임의 주제 선택 → 2) 나쁜 프롬프트 1개 작성 → 3) 본문 원칙으로 개선 → 4) 결과 비교
- 예상 결과물: 개선 전/후 프롬프트 1쌍 + 차이 설명 3줄
- 자가 평가 기준: 모호 표현 제거, 형식/제약 명시, 맥락·목적 포함

## 02장 질문 설계하기 — 실습 과제
- 난이도: [기본]
- 목표: 동일 문제를 질문 유형별로 재구성해 차이를 비교한다.
- 단계별 가이드: 과제 1개 선정 → 폐쇄형/개방형/비교형 버전 작성 → 답변 비교표 작성
- 예상 결과물: 3가지 질문과 답변 비교표(기준 3개)
- 자가 평가 기준: 유형 정의 적합성, 기준 명시, 재작업 감소 효과

## 03장 좋은 인스트럭션 — 실습 과제
- 난이도: [응용]
- 목표: 8가지 구성 요소를 모두 포함한 표준 인스트럭션을 작성한다.
- 단계별 가이드: 업무 선택 → 역할/목표/입·출력/제약/처리/성공 기준/톤 작성 → 검토
- 예상 결과물: 1페이지 인스트럭션 초안(MD)
- 자가 평가 기준: 성공 기준의 검증 가능성, 제약의 실행 가능성, 단계성 충족

## 04장 메타 원칙 — 실습 과제
- 난이도: [응용]
- 목표: SSOT·SoC를 적용해 중복을 제거한다.
- 단계별 가이드: 중복 규칙 2개 발견 → SSOT 파일로 분리 → 참조 구조 제안서 작성
- 예상 결과물: SSOT 초안(MD 또는 YAML) + 적용 지침 5줄
- 자가 평가 기준: 중복 제거 효과, 변경 영향 최소화, 참조 일관성

## 05장 에이전트·제약 — 실습 과제
- 난이도: [응용]
- 목표: 업무를 2–4개 워커 에이전트로 분해한다.
- 단계별 가이드: 책임 정의 → 역할/제약/출력 표 작성 → 안전장치(지식 출처/인용) 추가
- 예상 결과물: 에이전트 설계 표(역할/제약/출력)
- 자가 평가 기준: 단일 책임, 재사용성, 보안 제약 포함

## 06장 입출력 설계 — 실습 과제
- 난이도: [응용]
- 목표: 명세 기반 입·출력을 정의하고 샘플을 만든다.
- 단계별 가이드: 입력/출력 명세 문서화 → JSON Schema 작성 → 샘플 I/O 제작
- 예상 결과물: input.md, output.schema.json, sample.json
- 자가 평가 기준: 스키마 유효성, 후속 파싱 가능성, 성공 기준 매핑

## 07장 워크플로우 패턴 — 실습 과제
- 난이도: [응용] 🌿
- 목표: 파이프라인 3단계를 `workflow.yaml`로 정의한다.
- 단계별 가이드: 단계 정의 → 에이전트/입출력 지정 → 생성-검증 루프 추가
- 예상 결과물: workflows/your_flow/workflow.yaml
- 자가 평가 기준: 단계별 산출물 명확성, 의존성 적절성, 승인 분기 동작

## 07.5장 중간 규모 협업과 상태 관리 — 실습 과제
- 난이도: [중급-초반] 🌿
- 목표: 3-4개 에이전트 워크플로우를 설계하고 상태 추적을 구현한다.
- 단계별 가이드:
  1) 중간 규모 워크플로우 설계: 3-4단계 프로세스 정의 → workflow.yaml 작성 (재시도 정책 포함)
  2) 상태 추적 시스템: status.json 초기 상태 → 각 단계별 업데이트 시나리오 작성
  3) 디렉토리 구조 설계: /tasks/task-{id}/ 구조 → JSON 산출물 표준화
  4) 에러 처리 시나리오: 실패 시나리오 3가지 → 재시도 정책 → error.log 샘플
- 예상 결과물:
  - workflow.yaml (3-4단계, max_retries/timeout 포함)
  - status.json 초기 상태 및 업데이트 시나리오
  - 디렉토리 구조 설계 문서 (/tasks, /shared, /workflow.yaml)
  - 표준 JSON 산출물 샘플 (각 에이전트별)
  - error.log 샘플 (3가지 실패 시나리오)
- 자가 평가 기준:
  - 각 에이전트 역할 분리의 명확성
  - status.json이 모든 단계에서 정확히 업데이트되는가
  - JSON 산출물이 일관된 형식을 가지는가 (task_id, agent, status, timestamp, output)
  - 재시도 정책이 각 실패 시나리오에 적절한가
  - 다음 에이전트가 이전 산출물을 쉽게 사용할 수 있는가
- 심화 과제 (선택) 💎:
  - 7장 패턴(2개 에이전트)과 7.5장 패턴(4개 에이전트) 비교 분석
  - 10장 미리보기: 간단한 아키텍트 개념 적용 (아키텍트가 워커 선택하는 로직)

## 08장 성능 최적화 — 실습 과제
- 난이도: [응용]
- 목표: 단계적 모델 사용 전략을 수립하고 토큰 절감 리팩터를 수행한다.
- 단계별 가이드: 초안/검수 모델 분리 → 입력 20% 축약 → 품질 비교 기록
- 예상 결과물: 전략 노트 + 전/후 결과 비교표
- 자가 평가 기준: 품질 저하 최소화, 비용/시간 절감 수치화

## 09장 평가와 검증 — 실습 과제
- 난이도: [응용]
- 목표: A/B 실험과 오답 노트를 운영한다.
- 단계별 가이드: A/B 프롬프트 설계 → 표준 질문 세트 실행 → 점수 비교 → 오답 노트 업데이트
- 예상 결과물: 실험 리포트(MD) + 오답 노트 항목 3개
- 자가 평가 기준: 공정 비교(동일 조건), 회귀 방지 확인

## 10장 고급 아키텍처 — 실습 과제
- 난이도: [심화]
- 목표: 아키텍트 1 + 워커 3 구조를 설계하고 산출물 인터페이스를 정의한다.
- 단계별 가이드: 조직도/워크플로우 다이어그램 → 산출물 파일 구조(JSON/YAML) 명세
- 예상 결과물: 조직 설계(MD+Mermaid) + 인터페이스 스키마
- 자가 평가 기준: 역할 분리, 로그/상태 추적 가능성, 실패 처리 계획

## 13장 DSL — 실습 과제

## 14장 진화와 관리 — 실습 과제
