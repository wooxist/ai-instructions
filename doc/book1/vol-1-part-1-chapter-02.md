# 2장. 질문 설계하기

**Part 1: 프롬프트와 인스트럭션의 기초**

### 이 장에서 배우는 것
- AI의 답변을 원하는 방향으로 이끌기 위한 질문 설계 방법
- 상황별로 알맞은 질문 유형을 고르는 기준
- 유형별 바로 쓰는 템플릿과 예시·팁·함정
- 나쁜 질문을 좋은 질문으로 바꾸는 전/후 비교 방법

## 왜 질문 유형이 중요한가?

질문은 결과를 바꿉니다. 같은 사람, 같은 모델에게 물어도 ‘어떻게’ 묻느냐에 따라 전혀 다른 답이 돌아옵니다. 상황에 맞는 질문 유형을 의식적으로 선택하는 것은, 마치 목수가 작업에 맞는 연장을 고르는 것과 같습니다. 빠른 결정이 필요할 때는 망치(폐쇄형 질문)를, 섬세한 조각이 필요할 때는 조각칼(탐색형 질문)을 쓰는 것처럼, 질문의 유형을 잘 선택하면 불필요한 재작업을 줄이고 원하는 결과물을 더 빠르고 정확하게 얻을 수 있습니다.

이 장에서는 대표적인 6가지 질문 유형과 그 활용법을 살펴보겠습니다.

---

💡 **질문 유형 빠른 선택 가이드**

상황에 맞는 질문 유형을 빠르게 찾아보세요:

- **빠른 판단 필요?** → 폐쇄형 (예/아니오 답변)
- **깊이 있는 설명 필요?** → 개방형 (요약, 설명)
- **여러 아이디어 필요?** → 탐색형 (브레인스토밍)
- **대안 비교 필요?** → 비교형 (장단점 분석)
- **현실 제약 있음?** → 제약 기반형 (환경·예산 고려)
- **질문 자체가 애매함?** → 메타 질문 (질문 개선)

## 2.1 폐쇄형 질문 (Closed-ended)

**정의:** '예/아니오'처럼 답변의 범위가 명확하게 정해진 질문입니다. 빠른 판단이나 결정이 필요할 때, 특히 규정 준수 여부나 특정 기준 충족 여부를 확인할 때 매우 유용합니다.

**예시 1 (개발):** "이 PR은 보안 가이드라인 4항목(입력검증, 비밀번호 해시, 로그 마스킹, 비밀키 노출 금지)을 충족합니까? 예/아니오로 답하고 근거를 1줄로 설명해주세요."

**예시 2 (일반):** "이 프로젝트는 예산 승인 3가지 조건(ROI 15% 이상, 실행 기간 6개월 이내, 기존 시스템과 호환)을 충족합니까? 예/아니오로 답하고 각 조건별 충족 여부를 간략히 설명해주세요."

**팁과 함정:** "출시해도 돼?"나 "버그 없지?"와 같이 기준이 모호한 질문은 AI가 자의적으로 판단하게 만들어 답변의 신뢰도를 떨어뜨립니다. 좋은 폐쇄형 질문을 위해서는 판단의 근거가 되는 기준을 체크리스트 형태로 명확하게 제시하고, 답변에 대한 간단한 근거를 함께 요구하는 것이 좋습니다.

## 2.2 개방형 질문 (Open-ended)

**정의:** 특정 주제에 대해 설명, 요약, 서술을 요청하여 풍부한 정보를 얻을 때 사용합니다. 배경 설명이나 개념 정리, 사례 수집 등에 적합합니다.

**예시 1 (개발):** "임원 보고용으로 프로젝트 A의 경과를 5줄 불릿으로 요약해줘. 톤은 간결하고 결정지향적으로 하고, 마지막 줄에 다음 액션을 1개 포함해줘."

**예시 2 (일반):** "지난 분기 고객 불만 사항을 유형별로 분류하고, 상위 3가지 유형의 발생 빈도와 주요 원인을 설명해주세요. 각 유형당 2-3문장으로 작성해주세요."

**팁과 함정:** "현황 알려줘"처럼 범위를 너무 넓게 잡으면 불필요한 정보가 많아질 수 있습니다. 따라서 개방형 질문을 할 때는 답변의 대상, 기간, 자료원 등을 명시하여 범위를 좁히고, 원하는 결과물의 형식(표, 불릿 등)과 길이를 함께 지정하는 것이 중요합니다.

## 2.3 탐색형 질문 (Exploratory)

**정의:** 브레인스토밍처럼 다양한 아이디어나 대안을 모을 때 효과적입니다. AI의 창의성을 활용하여 새로운 가능성을 탐색하는 데 중점을 둡니다.

**예시:** "신규 사용자 온보딩 개선을 위한 아이디어 7개를 제안해주세요. 각 아이디어는 서로 다른 접근법(UI 개선, 콘텐츠 강화, 인센티브, 커뮤니티 등)을 사용하고, 구현 난이도는 1-5 점수로 표시해주세요."

**팁과 함정:** "아이디어 좀 줘"와 같은 막연한 질문은 실행 불가능한 아이디어를 낳기 쉽습니다. 따라서 탐색형 질문을 할 때는 원하는 아이디어의 개수, 평가 기준, 그리고 현실적인 제약 조건을 함께 명시하는 것이 중요합니다. '중복 금지'나 '다양한 카테고리' 같은 조건을 추가하면 더 풍부한 결과를 얻을 수 있습니다.

## 2.4 비교형 질문 (Comparative)

**정의:** 여러 대안을 명확한 기준에 따라 나란히 놓고 장단점을 분석하여 최적의 선택을 내릴 때 사용합니다. 도구나 기술 스택을 선택하거나, 아키텍처 대안을 비교할 때 특히 유용합니다.

**예시 1 (개발):** "LangChain과 LlamaIndex를 비용, 문서화, 커뮤니티, 확장성 기준으로 비교하는 표를 만들고, 최종 추천과 그 사유를 설명해줘."

**예시 2 (일반):** "원격근무와 사무실 근무를 비용, 생산성, 직원 만족도, 협업 효율성 4가지 기준으로 비교하는 표를 만들고, 우리 팀에 추천하는 근무 방식과 이유를 설명해주세요."

**팁과 함정:** "뭐가 더 좋아?"와 같은 질문은 기준이 없기 때문에 의미 있는 답변을 얻기 어렵습니다. 좋은 비교형 질문을 위해서는 3~5개의 명확한 비교 기준을 제시하고, 각 대안의 장단점(Trade-off)[^2]을 함께 분석하도록 요청하는 것이 효과적입니다.

## 2.5 제약 기반형 질문 (Constraint-based)

**정의:** 시간, 예산, 기술 스택, 정책 등 **현실적인 제약 조건이 핵심인 질문**입니다. 단순한 이상적 해결책이 아닌 '실행 가능한' 해결책을 얻고자 할 때 사용합니다.

**예시 1 (개발):** "Python 3.10, MacOS, 인터넷 제한 환경에서 주 2시간만 투입 가능할 때, 데이터 정제 자동화의 최소 실행 단계를 제안해줘."

**예시 2 (일반):** "예산 500만원, 실행 기간 2주, 내부 인력만 활용 가능한 상황에서 신규 직원 온보딩 프로그램을 개선하려고 합니다. 가장 효과적인 3단계 실행 계획을 제시해주세요."

**팁과 함정:** 제약 조건이 빠진 질문은 현실과 동떨어진 '책상머리 설계'로 이어지기 쉽습니다. 따라서 질문의 맨 앞에 환경 정보를 명시하고, AI가 어떤 가정을 기반으로 답변했는지 명시하도록 하여 오해의 소지를 줄이는 것이 좋습니다.

## 2.6 메타 질문 - 질문을 개선하는 질문 (Meta-questions)

**정의:** 질문 자체에 대해 질문하는 것입니다. 문제 정의가 모호하거나 더 나은 질문을 만들고 싶을 때 사용하여, 다음 질문의 품질을 높이는 역할을 합니다.

**예시:** "내가 하려는 질문을 더 효과적으로 만들기 위해, 누락되거나 모호한 부분을 지적하고 개선안을 제시해줘. 개선 전과 후를 나란히 비교해줘."

**팁과 함정:** 단순히 "이 질문 어때?"라고 묻기보다는, '개선 전/후 비교'를 요구하거나 '목적, 대상, 범위, 형식, 성공 기준'의 5가지 요소에 맞춰 질문을 재구성하도록 요청하면 훨씬 더 구체적인 도움을 받을 수 있습니다.

이처럼 상황에 맞는 질문 유형을 선택하고 설계하는 것은 효과적인 인스트럭션의 첫걸음입니다. 좋은 질문은 좋은 답변을 이끌어내지만, 반복적인 작업을 자동화하고 팀 전체의 생산성을 높이기 위해서는 일회성 질문을 넘어 체계적인 '지시 체계'가 필요합니다.

이제 좋은 질문을 바탕으로, 일관되고 예측 가능한 결과를 만들어내는 지시 작성의 기본 원칙들을 [3장. 효과적인 지시 작성의 기본 원칙](vol-1-part-1-chapter-03.md)에서 본격적으로 살펴보겠습니다.

## 요약

이 장에서는 AI와 효과적으로 소통하기 위한 6가지 질문 유형을 배웠습니다. 상황에 맞는 질문 유형을 선택하는 것은 원하는 결과를 얻는 첫걸음입니다.

**핵심 개념:**
좋은 질문은 좋은 답변을 만듭니다. 질문 유형은 도구와 같아서, 상황에 맞는 도구를 선택해야 효율적인 결과를 얻을 수 있습니다.

**6가지 질문 유형:**
- **폐쇄형**: 빠른 판단이 필요할 때 (예/아니오, 체크리스트 기반)
- **개방형**: 깊이 있는 설명이나 요약이 필요할 때
- **탐색형**: 여러 아이디어를 모으거나 브레인스토밍할 때
- **비교형**: 여러 대안을 기준에 따라 비교할 때
- **제약 기반형**: 시간, 예산, 기술 등 현실 제약이 있을 때
- **메타 질문**: 질문 자체를 개선하고 싶을 때

**다음 단계:**
이제 좋은 질문을 바탕으로 일관되고 재사용 가능한 '인스트럭션'을 만드는 방법을 [3장. 효과적인 지시 작성의 기본 원칙](vol-1-part-1-chapter-03.md)에서 배워보겠습니다. 일회성 질문을 넘어 팀 전체가 활용할 수 있는 지시 체계로 발전시키는 방법을 알아봅니다.

### 이 장을 완료하셨다면 다음을 확인하세요:

**개념 이해:**
- [ ] 6가지 질문 유형(폐쇄형, 개방형, 탐색형, 비교형, 제약 기반형, 메타)의 차이를 설명할 수 있다
- [ ] 각 질문 유형이 적합한 상황을 구분할 수 있다
- [ ] 좋은 질문과 나쁨 질문의 차이를 설명할 수 있다

**실습 능력:**
- [ ] 상황에 맞는 질문 유형을 선택할 수 있다
- [ ] 폐쇄형 질문에 판단 기준을 명확히 제시할 수 있다
- [ ] 개방형 질문의 범위를 적절하게 조절할 수 있다
- [ ] 비교형 질문에 3-5개의 명확한 기준을 제시할 수 있다

**적용 및 활용:**
- [ ] 막연한 질문을 구체적인 질문으로 개선할 수 있다
- [ ] 메타 질문을 활용하여 질문 품질을 향상시킬 수 있다
- [ ] 제약 조건을 질문에 포함하여 현실적인 답변을 유도할 수 있다

### 실습 과제

**과제 1: 질문 유형 변환**

다음 질문을 6가지 유형으로 각각 변환하고, 어떤 상황에 각 유형이 적합한지 설명하세요:

*원본:* "이 기능을 추가할까요?"

1. 폐쇄형으로: (예: "사용자 수, 구현 비용, 기술 난이도 3가지 기준으로 볼 때 추가해야 하나요?")
2. 개방형으로:
3. 탐색형으로:
4. 비교형으로:
5. 맥락 의존형으로:
6. 메타 질문으로:

**과제 2: 나쁨 질문 개선하기**

다음 나쁨 질문들을 해당 유형의 팁을 활용하여 개선하세요:

1. 폐쇄형: "출시해도 돼?" → 기준을 명확히 하여 개선
2. 개방형: "현황 알려줘" → 범위를 좁혀서 개선
3. 탐색형: "아이디어 좀 줘" → 개수, 기준, 제약 명시하여 개선
4. 비교형: "뭐가 더 좋아?" → 비교 기준 제시하여 개선

**과제 3: 실전 질문 설계**

자신의 업무에서 다음 상황에 맞는 질문을 설계하세요:

1. **빠른 결정이 필요한 상황**: 폐쇄형 질문 1개
2. **브레인스토밍이 필요한 상황**: 탐색형 질문 1개
3. **여러 대안 비교가 필요한 상황**: 비교형 질문 1개

각 질문에 해당 유형의 핵심 요소(판단 기준, 비교 기준, 제약 조건 등)가 포함되어 있는지 확인하세요.
