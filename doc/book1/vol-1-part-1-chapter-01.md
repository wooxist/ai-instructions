# 1장. 좋은 프롬프트의 원칙

**Part 1: 프롬프트와 인스트럭션의 기초**

### 이 장에서 배우는 것
- 프롬프트와 인스트럭션의 기본 개념
- 좋은 프롬프트를 작성하는 법과 그 한계
- 프롬프트를 잘 쓰기 위한 핵심 요소(명확성·구체성·맥락·형식·제약)
- 프롬프트 사용 방식의 차이와 인스트럭션의 필요성

## 1.1 프롬프트란?

[프롬프트(Prompt)](glossary.md#프롬프트-prompt)는 AI 모델에게 말을 거는 방식이자, 우리가 원하는 것을 얻어내기 위한 '지시문'입니다. 이는 마치 우리가 새로운 외국어를 배우듯, AI라는 새로운 지적 파트너와 소통하는 방법을 배우는 것과 같습니다. 단순한 질문부터 복잡한 요청, 상세한 설명과 예시까지 다양한 형태를 가질 수 있습니다.

## 1.2 프롬프트를 잘 사용하려면

효과적인 프롬프트는 AI가 사용자의 의도를 정확히 파악하고 원하는 결과물을 만들어낼 수 있도록 돕는 '좋은 질문'입니다. 좋은 프롬프트를 작성하기 위해서는 다섯 가지 핵심 요소를 갖추어야 합니다.

### 1) 명확성과 구체성

가장 기본이 되는 것은 **명확성**과 **구체성**입니다. 단순히 "기후 변화에 대해 알려줘"라고 묻기보다 "최근 10년간 해수면 상승에 영향을 미친 3가지 요점을 설명해줘"처럼 구체적으로 질문하면 훨씬 더 유용한 답변을 얻을 수 있습니다.

### 2) 맥락과 목적

여기에 **맥락**과 **목적**을 더하면 결과물의 품질을 더 높일 수 있습니다. AI가 사용자의 상황을 이해하고 작업의 최종 목표를 알 때, 더 적절한 결과물을 생성하기 때문입니다. "이 코드를 개선해줘"라는 요청보다는 "저는 파이썬 초보 개발자입니다. 아래 코드는 분석 속도가 너무 느린데, 실행 속도를 개선하고 그 이유를 1~2줄로 설명해주세요."와 같이 배경 정보와 목적을 함께 제공하는 것이 좋습니다.

### 3) 예시와 역할

원하는 결과물의 **예시**를 한두 개 제공하는 것도 AI가 사용자의 의도를 파악하는 데 큰 도움이 됩니다. 이는 AI가 소수의 예시만으로도 패턴을 학습하여 새로운 문제를 해결하는 'Few-shot learning'[^1] 원리를 활용하는 것입니다. 또한, AI에게 특정 전문가의 **역할**을 부여하면(예: "당신은 데이터 과학자입니다."), 답변의 톤과 관점을 원하는 방향으로 유도할 수 있습니다.

### 4) 형식과 제약 조건

마지막으로, 결과물의 **형식**을 명확히 지정하고(예: "마크다운 표 형식으로"), 지켜야 할 **제약 조건**을 명시하면(예: "300단어 이내로"), AI는 사용자가 원하는 구조에 맞춰 예측 가능한 결과물을 생성하게 됩니다.

### 5) 피해야 할 표현

반대로, 좋은 프롬프트 작성을 방해하는 표현들도 있습니다. "충분히", "적절히"와 같이 기준이 모호한 형용사를 피하고, 구체적인 수량이나 기준을 명시하는 것이 좋습니다. 또한, 작업의 목적과 범위를 명확히 하고, 원하는 결과물의 형식과 성공 기준을 제시해야 AI가 방황하지 않고 정확한 결과물을 만들어낼 수 있습니다.

이러한 요소들은 2장과 3장에서 질문 설계와 인스트럭션 템플릿을 통해 더 자세히 다룰 것입니다.

## 요약

이 장에서는 AI와의 효과적인 소통을 위한 **좋은 프롬프트의 5가지 핵심 원칙**을 배웠습니다.

- **명확성과 구체성**: 모호함 없이 정확하게 지시하기
- **맥락과 목적**: AI가 배경 상황을 이해하도록 돕기
- **예시와 역할**: 원하는 결과물의 모습과 AI의 관점을 지정하기
- **형식과 제약 조건**: 결과물의 구조를 통제하기
- **피해야 할 표현**: 모호한 형용사 대신 구체적인 기준 제시하기

이 원칙들은 모든 AI와의 소통에 적용되는 기본기입니다.

다음 장에서는 이 원칙들, 특히 '명확성과 구체성'을 **'좋은 질문'**이라는 구체적인 기술로 심화시키는 방법을 배우겠습니다.

## 참고: 주요 프롬프팅 기법들

AI와의 상호작용은 단순한 질문-답변을 넘어 다양한 기법으로 발전해왔습니다. 대표적인 프롬프팅 기법들은 다음과 같습니다:

### Zero-shot 프롬프팅
예시 없이 바로 작업을 요청하는 방식입니다.
- 예: "번역해줘: Hello world"
- 특징: 가장 간단하지만 복잡한 작업에는 한계가 있음

### Few-shot 프롬프팅
AI에게 몇 가지 예시를 먼저 제공하여 패턴을 학습시키는 방식입니다.
- 예: "예시: 사과→apple, 바나나→banana. 그럼 오렌지는?"
- 특징: 패턴이 명확한 작업에 효과적

### Chain-of-Thought (CoT) 프롬프팅
AI가 단계별로 사고 과정을 보여주도록 유도하는 방식입니다.
- 예: "이 문제를 단계별로 생각해보자. 먼저..."
- 특징: 복잡한 추론이나 수학 문제 해결에 유용

### Role 프롬프팅
AI에게 특정 전문가나 역할을 부여하는 방식입니다.
- 예: "당신은 10년 경력의 데이터 과학자입니다."
- 특징: 답변의 톤, 관점, 전문성 수준을 조정할 수 있음

이러한 기법들은 상황에 따라 선택적으로 활용되며, 이후 인스트럭션 설계에서도 핵심 구성 요소가 됩니다.

## 실습 체크리스트

**개념 이해:**
- [ ] 프롬프트가 무엇인지 설명할 수 있다
- [ ] 프롬프트의 5가지 핵심 요소(명확성, 구체성, 맥락, 형식, 제약)를 이해했다
- [ ] 주요 프롬프팅 기법들(Zero-shot, Few-shot, CoT, Role)의 차이를 설명할 수 있다

**실습 능력:**
- [ ] 간단한 프롬프트에 5가지 핵심 요소를 추가하여 개선할 수 있다
- [ ] 역할(Role) 부여와 Few-shot 예시를 포함한 프롬프트를 작성할 수 있다
- [ ] 모호한 표현을 피하고 구체적인 기준을 명시할 수 있다

**적용 및 활용:**
- [ ] 자신의 업무에서 각 프롬프팅 기법을 적절히 선택하여 활용할 수 있다
- [ ] 단순 프롬프트를 5가지 핵심 요소로 체계적으로 개선할 수 있다

### 실습 과제

**과제 1: 5가지 핵심 요소로 프롬프트 개선하기**

다음 단순한 프롬프트를 5가지 핵심 요소를 모두 포함하도록 개선하세요:

*원본:* "회의록을 요약해줘"

*개선 방향:*
1. 명확성과 구체성: 어떤 회의록인지, 요약 길이와 포함할 항목
2. 맥락과 목적: 요약의 목적, 누가 사용할지
3. 예시와 역할: (필요시) 원하는 요약 스타일 예시
4. 형식과 제약 조건: 결과물 구조, 반드시 지켜야 할 조건
5. 피해야 할 표현: 모호한 표현 대신 구체적 기준 사용

**과제 2: 프롬프팅 기법 비교**

"이메일 분류" 작업에 대해 다음 3가지 방식으로 프롬프트를 작성하고 결과를 비교하세요:
1. Zero-shot: 예시 없이 바로 요청
2. Few-shot: 2-3개 예시 포함
3. Role 프롬프팅: 전문가 역할 부여

*기대 효과:* 각 기법의 특성과 적절한 사용 상황을 체감하기

**과제 3: 자주 쓰는 프롬프트 개선하기**

자신이 자주 사용하는 프롬프트 하나를 선택하여 개선하세요:

1. **현재 프롬프트**: 지금 사용 중인 프롬프트를 그대로 작성하세요

2. **5가지 요소 체크**:
   - 명확성/구체성: ☐ 충분 / ☐ 부족 (무엇이 부족한가?)
   - 맥락/목적: ☐ 충분 / ☐ 부족 (무엇이 부족한가?)
   - 예시/역할: ☐ 포함 / ☐ 불포함 (필요한가?)
   - 형식: ☐ 명시 / ☐ 불명확 (어떤 형식을 원하는가?)
   - 제약 조건: ☐ 명시 / ☐ 불명확 (어떤 제약이 필요한가?)

3. **개선된 프롬프트**: 부족한 요소를 보완하여 새 버전 작성

4. **기대 효과**: 개선 전후 결과물의 차이를 예상해보세요
