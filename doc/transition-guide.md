## 📚 1권 핵심 개념 복습

### 1권에서 배운 것

2권을 시작하기 전에 1권의 핵심 개념을 빠르게 복습하겠습니다.

#### Part 1: 프롬프트와 인스트럭션 (Ch 1-3)

**핵심 개념**:
- **프롬프트**: 일회성 AI 대화
- **인스트럭션**: 재사용 가능한 구조화된 명세서
- **8가지 구성 요소**: 역할, 목표, 입력, 출력, 제약, 예시, 프로세스, 품질

**2권 연결**:
```
프롬프트 → 인스트럭션 (1권)
    ↓
목표 → 사고 클러스터 (2권)
```

#### Part 2: 설계 원칙 (Ch 4-6)

**핵심 개념**:
- **단일 책임 원칙**: 하나의 에이전트, 하나의 역할
- **메타 원칙**: 구조적, 실행, 검증
- **명확한 입출력**: 예측 가능한 결과

**2권 연결**:
```
단일 에이전트 (1권)
    ↓
사고 조율자 + 실행 워커 (2권)
```

#### Part 3: 워크플로우와 평가 (Ch 7-9)

**핵심 개념**:
- **3가지 워크플로우**: 파이프라인, 생성-검증, 라우팅
- **성능 최적화**: 품질-비용-속도 트레이드오프
- **평가 지표**: 정량적, 정성적 측정

**2권 연결**:
```
기본 워크플로우 (1권)
    ↓
사고 워크플로우 + 피드백 루프 (2권)
```

#### Part 4: 실전 패턴 (Ch 10)

**핵심 개념**:
- **복잡도별 설계**: 간단/표준/복잡
- **단일 에이전트 패턴**: 이메일, 마케팅, 회의록

**2권 연결**:
```
단일 에이전트 설계 (1권)
    ↓
다중 에이전트 협업 시스템 (2권)
```

## 🚀 2권 사전 준비

### 2권에서 다루는 것

2권은 **AI 사고 생태계**를 중심으로 조직 수준의 설계를 다룹니다.

#### 핵심 프레임워크: 6계층 구조

```
┌─────────────────────────────────────────┐
│  1. 미션 (Mission)                       │  ← 철학적 기반
├─────────────────────────────────────────┤
│  2. 핵심 가치 (Core Values)              │  ← 의사결정 기준
├─────────────────────────────────────────┤
│  3. 비전 (Vision)                        │  ← 미래 상태
├─────────────────────────────────────────┤
│  4. 목표 (Goals)                         │  ← 전략적 방향
├─────────────────────────────────────────┤
│  5. 사고 클러스터 (Thinking Clusters)    │  ← 2권 핵심
├─────────────────────────────────────────┤
│  6. 실행 항목 (Execution Items)          │  ← 구체적 행동
└─────────────────────────────────────────┘
```

**2권 초점**: 5층(사고 클러스터) 설계 및 피드백 루프

### 새로운 핵심 개념

#### 1. 사고 클러스터 (Thinking Cluster)

**정의**: 복잡한 목표를 관리 가능한 사고 단위로 분해한 것

**1권과의 차이**:
```yaml
vol1:
  focus: "단일 에이전트가 무엇을 하는가"
  example: "이메일 요약 에이전트"
  
vol2:
  focus: "여러 사고 단위가 어떻게 협업하는가"
  example: "마케팅 캠페인 사고 클러스터 (기획 + 콘텐츠 + 디자인)"
```

#### 2. 사고 조율자 vs 실행 워커

**사고 조율자** (인간):
- 전체 사고 흐름 설계
- 맥락적 판단
- 전략적 의사결정

**실행 워커** (AI):
- 정의된 사고 프로세스 실행
- 데이터 분석 및 패턴 인식
- 반복 작업 수행

**1권 개념 확장**:
```
1권: 단일 에이전트 역할 정의
    ↓
2권: 역할 분담 (조율자 + 워커)
```

#### 3. 피드백 루프

**정의**: 실행 결과가 사고와 전략으로 되돌아가는 구조

```
목표 → 사고 클러스터 → 실행
         ↑               ↓
         └─── 피드백 ─────┘
```

**1권과의 차이**:
```yaml
vol1:
  evaluation: "단일 에이전트 성능 평가"
  
vol2:
  learning: "조직 전체의 지속적 학습"
```

## ❓ 자주 묻는 질문 (FAQ)

### Q1: 1권을 읽지 않고 2권부터 시작해도 되나요?

**A**: 가능하지만 권장하지 않습니다.

**이유**:
- 2권은 1권의 개념(에이전트, 워크플로우 등)을 확장합니다
- 기초 없이 시작하면 어려울 수 있습니다

**대안**:
- **조직 설계에 급한 분**: Preface 필독 + 필요시 1권 참조
- **시간 여유 있는 분**: 1권 → 2권 순서 학습

### Q2: 1권과 2권의 가장 큰 차이는 무엇인가요?

**A**: 범위와 복잡도입니다.

```yaml
vol1:
  scope: "개인, 소규모 팀"
  focus: "단일 에이전트 설계"
  difficulty: "★☆☆ ~ ★★☆"
  
vol2:
  scope: "팀, 조직"
  focus: "다중 에이전트 협업 시스템"
  difficulty: "★★☆ ~ ★★★"
```

### Q3: 2권에서 가장 중요한 개념은 무엇인가요?

**A**: **사고 클러스터**와 **피드백 루프**입니다.

**사고 클러스터**:
- 복잡한 문제를 관리 가능한 사고 단위로 분해
- 인간과 AI의 협업 구조

**피드백 루프**:
- 실행 결과를 학습으로 전환
- 지속적 개선의 핵심

### Q4: 1권의 어떤 개념이 2권에서 가장 많이 사용되나요?

**A**: **에이전트 역할 설계**(Ch 5)와 **워크플로우 패턴**(Ch 7)입니다.

**활용 예**:
```
1권 Ch 5 "에이전트 역할"
    ↓
2권: 사고 조율자 + 실행 워커

1권 Ch 7 "워크플로우 패턴"
    ↓
2권: 4 Stage 사고 프로세스
```

### Q5: 2권을 다 읽으면 무엇을 할 수 있나요?

**A**: 조직 수준의 AI 사고 시스템을 설계하고 운영할 수 있습니다.

**구체적으로**:
- ✅ 복잡한 목표를 사고 클러스터로 분해
- ✅ 여러 AI 에이전트를 조율
- ✅ 피드백 루프로 지속적 학습
- ✅ 학습하는 조직 구축

### Q6: 2권 학습에 필요한 시간은?

**A**: 약 30-40시간 (실습 포함)

**세부**:
- 이론 학습: 20시간
- 실습 과제: 10-20시간
- 복습 및 적용: 개인차

### Q7: 2권에서 코딩이 필요한가요?

**A**: 일부 실습에서 선택적으로 필요합니다.

**코딩이 필요한 경우**:
- Ch 13: 코드 실행 도구, API 호출
- 심화 실습: Python 자동화 스크립트

**코딩 없이도 가능**:
- 개념 이해와 설계는 코딩 없이 가능
- YAML/JSON 명세서 작성으로 대체 가능

### Q8: 1권과 2권 중 어느 것이 더 어렵나요?

**A**: 2권이 더 어렵지만, 1권을 완료했다면 충분히 소화 가능합니다.

**난이도 비교**:
```
1권 최고 난이도: ★★☆ (Part 4)
2권 시작 난이도: ★★☆ (Ch 11)
2권 최고 난이도: ★★★ (Ch 12, 14, 15)
```
