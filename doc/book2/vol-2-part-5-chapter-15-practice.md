# 15장 실습 과제 (상세 가이드)

이 장에서 배운 피드백 루프와 지속적 학습을 실제로 적용해보는 실습입니다.

## 기본 실습

### 실습 1: 결과 수집 체계 설계

**목표**: 체계적인 피드백 데이터 수집 시스템을 구축합니다.

**과제**:
1. 측정 지표 정의 (1시간)
   - 정량 지표 3-5개 선택
     - 예: 완료율, 처리 시간, 비용, 품질 점수
   - 정성 피드백 채널 2-3개 설정
     - 예: 설문, 회고 미팅, 1:1 인터뷰
   - 각 지표의 목표값 설정

2. 수집 도구 선택 (30분)
   - 로그 시스템: Elasticsearch / 파일 로그
   - 설문 도구: Google Forms / Typeform
   - 대시보드: Grafana / 자체 개발

3. 수집 프로세스 설계 (30분)
   - 수집 주기 결정 (실시간 / 일간 / 주간)
   - 책임자 지정 (누가 수집하고 정리하는가)
   - 저장 위치 및 형식

4. 2주간 파일럿 실행 (2주)
   - 실제로 데이터 수집
   - 샘플 데이터 검증
   - 프로세스 조정

**평가 기준**:
- 측정 지표가 SMART한가? (Specific, Measurable, Achievable, Relevant, Time-bound)
- 수집 프로세스가 지속 가능한가?
- 데이터 품질이 분석에 충분한가?

**제출 형식**:
- 측정 지표 문서 (metrics.yaml)
- 수집 프로세스 가이드 (collection_guide.md)
- 2주간 샘플 데이터
- 프로세스 개선 사항

**예상 소요 시간**: 2시간 설계 + 2주 실행

### 실습 2: 패턴 분석 및 인사이트 도출

**목표**: AI와 협업하여 피드백 데이터에서 의미있는 인사이트를 도출합니다.

**과제**:
1. 데이터 준비 (30분)
   - 수집된 데이터를 CSV/JSON으로 정리
   - 결측치 처리
   - 기본 통계 계산 (평균, 표준편차 등)

2. AI 패턴 분석 (1.5시간)
   - 상관관계 분석
     - 예: "특정 Stage에서 시간이 오래 걸리면 품질이 낮은가?"
   - 시간별 추세 분석
     - 예: "주말 작업의 품질이 평일보다 낮은가?"
   - 이상치 탐지
     - 예: "갑자기 오류율이 3배 증가한 날"
   
   도구: Python (pandas, matplotlib) 또는 Claude/ChatGPT

3. 인간 해석 (1.5시간)
   - 각 패턴에 비즈니스 맥락 부여
   - 인과관계 추론
   - 영향도 평가 (High / Medium / Low)
   - 개선 기회 식별 (3-5개)

4. 인사이트 보고서 작성 (30분)
   - 핵심 발견 사항 (3-5개)
   - 각 발견의 근거 (데이터)
   - 권장 조치
   - 우선순위

**평가 기준**:
- 패턴 분석이 데이터 기반인가?
- 인사이트가 실행 가능한가?
- 비즈니스 영향이 명확한가?

**제출 형식**:
- 데이터 분석 Jupyter Notebook
- 시각화 차트 (5-10개)
- 인사이트 보고서 (insights_report.md)
- 액션 아이템 리스트

**예상 소요 시간**: 4시간

### 실습 3: 조정 계획 수립

**목표**: 인사이트를 바탕으로 3가지 수준의 조정 계획을 수립합니다.

**과제**:
1. 조정 수준 결정 (30분)
   - 각 인사이트에 적절한 조정 수준 선택
     - Level 1 (실행 항목): 빠르고 영향 제한적
     - Level 2 (사고 클러스터): 중간 노력, 구조적 변경
     - Level 3 (목표): 큰 노력, 전략적 변경
   
   의사결정 매트릭스:
     - 영향도 vs 실행 난이도
     - Quick Wins 우선 (높은 영향 + 낮은 난이도)

2. Level 1 조정 (1시간)
   - 프롬프트 개선
   - 파라미터 튜닝
   - 검증 규칙 강화
   - 즉시 적용 가능한 변경

3. Level 2 조정 (1시간)
   - 사고 Stage 추가/제거
   - 워커 역할 재정의
   - Human-in-the-Loop 지점 조정
   - 2-4주 계획 필요

4. Level 3 조정 (30분)
   - 목표 수정 또는 새 목표 추가
   - 리소스 재배분
   - 전략적 방향 전환
   - 리더십 승인 필요

**평가 기준**:
- 조정 계획이 인사이트와 연결되는가?
- 실행 가능성이 있는가?
- 예상 효과가 구체적인가?

**제출 형식**:
- 조정 계획서 (adjustment_plan.md)
- Level별 구체적 실행 계획
- 승인 프로세스 문서
- 일정 및 마일스톤

**예상 소요 시간**: 3시간

### 실습 4: 목표 충돌 분석

**목표**: 조직의 모든 목표를 분석하여 충돌을 해결합니다.

**과제**:
1. 목표 인벤토리 (1시간)
   - 조직의 모든 목표 나열 (5-15개)
   - 각 목표의 KPI, 리소스, 책임자 명시
   - 목표 간 관계 초기 파악

2. 충돌 탐지 (1.5시간)
   
   **방법 1: AI 도구 활용**
   - 의미론적 유사도 계산 (중복 탐지)
   - 자원 경쟁 분석 (상충 탐지)
   - 입출력 매칭 (의존 탐지)
   
   **방법 2: 수동 체크리스트**
   - 상충 체크: 반대 방향 추구?
   - 중복 체크: 같은 결과 추구?
   - 의존 체크: 순서 제약 있는가?

3. 해결 전략 수립 (1시간)
   
   각 충돌 유형별:
   - **상충**: 핵심 가치 우선순위로 조정
   - **중복**: 목표 병합 또는 재정의
   - **의존**: 실행 순서 최적화

4. 조정된 목표 명세 (30분)
   - 병합/수정/삭제된 목표 명시
   - 새로운 목표 간 관계 다이어그램
   - 승인 및 커뮤니케이션 계획

**평가 기준**:
- 모든 유형의 충돌을 식별했는가?
- 해결 전략이 핵심 가치와 일치하는가?
- 조정된 목표가 실행 가능한가?

**제출 형식**:
- 목표 인벤토리 (goals_inventory.yaml)
- 충돌 분석 보고서 (conflicts_analysis.md)
- 해결 전략 문서
- 조정된 목표 명세

**예상 소요 시간**: 4시간

### 실습 5: Blameless Postmortem 작성

**목표**: 실패를 학습 기회로 전환하는 문서를 작성합니다.

**과제**:
1. 사건 선정 및 타임라인 작성 (1시간)
   - 최근 실패 사례 1개 선택
   - What happened? (무슨 일이 있었나)
     - 예: 14:30 배포 → 14:45 첫 오류 → 16:00 롤백
   - 영향 범위 및 비용 측정

2. 근본 원인 분석 (1시간)
   
   **방법: 5 Whys 기법**
   - Why 1: 왜 오류가 발생했나?
   - Why 2: 왜 테스트에서 발견 못했나?
   - Why 3: 왜 테스트 커버리지가 부족했나?
   - Why 4: 왜 프로세스에 테스트가 없었나?
   - Why 5: 왜 프로세스가 정의되지 않았나?
   
   분류: 직접 원인 vs 근본 원인

3. 학습 사항 정리 (30분)
   
   3가지 차원:
   - 기술적 교훈
   - 프로세스 교훈
   - 조직적 교훈

4. 예방 조치 계획 (30분)
   - 구체적 Action Items (5-10개)
   - 각 아이템의 책임자 및 기한
   - 우선순위 설정

5. 팀 공유 및 피드백 (1시간 회의)
   - Postmortem 발표
   - 팀원 의견 수렴
   - Action Items 조정
   - 감사 표현 (Appreciations)

**평가 기준**:
- 비난 없이 사실 중심으로 작성했는가?
- 근본 원인까지 파악했는가?
- 예방 조치가 구체적이고 실행 가능한가?

**제출 형식**:
- Blameless Postmortem 문서
- Action Items 트래킹 시트
- 회의 기록
- 실패 사례 DB 등록

**예상 소요 시간**: 3시간 + 회의 1시간

### 실습 6: PDCA 사이클 실행

**목표**: 한 가지 개선을 4주에 걸쳐 완전히 실행합니다.

**과제**:

**Week 1 - Plan (계획)**:
- 개선 목표 선정 (구체적, 측정 가능)
- 가설 수립 (If-Then 형식)
- 성공 기준 정의 (3-5개 지표)
- 실험 설계 (파일럿 범위, 기간)

**산출물**: PDCA Plan 문서

**Week 2-3 - Do (실행)**:
- 파일럿 실행 (소규모, 통제된 환경)
- 실시간 모니터링
- 데이터 수집 (정량 + 정성)
- 문제 발생 시 즉시 기록

**산출물**: 실행 로그, 데이터

**Week 4 - Check (검증)**:
- 데이터 분석 (가설 검증)
- 성공 기준 달성 여부 평가
- 예상치 못한 결과 분석
- 트레이드오프 평가

**산출물**: 분석 보고서

**Week 4 말 - Act (조치)**:
- 결정: 확대 / 조정 / 폐기
- 확대 시: 롤아웃 계획 수립
- 조정 시: 다음 PDCA 계획
- 폐기 시: 학습 사항 정리

**산출물**: 최종 결정 문서

**평가 기준**:
- PDCA 4단계를 모두 완료했는가?
- 데이터 기반 의사결정인가?
- 학습 사항이 문서화되었는가?

**제출 형식**:
- 완전한 PDCA 사이클 보고서 (20-30페이지)
- 데이터 및 분석
- 학습 사항
- 다음 사이클 계획

**예상 소요 시간**: 4주 (주당 2-4시간)

## 심화 실습

### 실습 7: 학습 문화 진단 및 개선 프로젝트

**목표**: 팀/조직의 학습 문화를 진단하고 개선 계획을 수립합니다.

**과제**:

**Phase 1 (Week 1-2): 진단**
- 학습 문화 설문 (전체 팀원)
- 1:1 인터뷰 (5-10명)
- 현재 수준 평가 (5개 차원)
  1. 심리적 안전감
  2. 피드백 개방성
  3. 실험 문화
  4. 지식 공유
  5. 지속적 개선

**Phase 2 (Week 3-4): 개선 계획**
- 우선순위 영역 선정 (2-3개)
- 구체적 실행 계획
  - 예: 주간 회고 도입, 실패 공유 세션, 학습 시간 확보
- 리더십 참여 확보

**Phase 3 (Week 5-6): 실행**
- 파일럿 프로그램 운영
- 참여 독려 및 지원
- 중간 점검 및 조정

**Phase 4 (Week 7-8): 평가**
- 재진단 (동일 설문)
- 개선 효과 측정
- 전체 롤아웃 계획

**제출 형식**:
- 진단 보고서
- 개선 계획서
- 파일럿 결과 보고서
- 롤아웃 로드맵

**예상 소요 시간**: 8주

### 실습 8: 실패 사례 데이터베이스 구축

**목표**: 조직의 실패를 체계적으로 기록하고 학습 자산화합니다.

**과제**:

**Week 1: 데이터베이스 설계**
- 실패 사례 스키마 정의
  - 필드: 제목, 날짜, 카테고리, 심각도, 근본 원인, 교훈, 예방 조치
- 도구 선택 (Notion / Confluence / 자체 개발)
- 검색 및 태깅 시스템

**Week 2: 초기 데이터 수집**
- 과거 실패 사례 5-10개 등록
- Blameless Postmortem 작성
- 팀원 교육 (등록 방법)

**Week 3: 활용 메커니즘 구축**
- 유사 사례 검색 기능
- 정기 리뷰 프로세스
- 신규 프로젝트 시 사례 참고 의무화

**Week 4: 출시 및 홍보**
- 전사 공지
- 사용 가이드 배포
- 인센티브 제도 (등록/활용)

**제출 형식**:
- 실패 사례 데이터베이스 (최소 10개 사례)
- 운영 가이드
- 활용 프로세스 문서
- 월간 리포트 템플릿

**예상 소요 시간**: 4주
