## 14.5 실습 과제

이 장에서 배운 완전한 실전 사례를 자신의 프로젝트에 적용해보는 실습입니다.

### 기본 실습

#### 실습 1: 사례 연구 분석

**목표**: 14.1 마케팅 콘텐츠 생성 사례를 깊이 이해합니다.

**과제**:
1. 14.1을 정독하세요 (1시간)
2. 6계층 구조를 Mermaid로 다이어그램화하세요:
   - 1층(미션) → 6층(파일 시스템) 연결
   - 핵심 가치가 어떻게 적용되는지 표시
3. 각 Stage별 사고 흐름을 요약하세요:
   - Stage 1: 어떤 활동, 어떤 산출물
   - Stage 2: 핵심 가치 적용 방법
   - Stage 3: 초안 작성 과정
   - Stage 4: 품질 검증 기준
4. 파일 시스템 구조를 매핑하세요:
   - 각 Stage의 산출물이 어느 파일에 저장되는가
   - thinking_state.json은 언제 업데이트되는가
5. 피드백 루프를 추적하세요:
   - 실행 결과가 어떻게 사고 개선으로 이어지는가

**평가 기준**:
- 6계층의 흐름을 명확히 이해하는가?
- 사고 → 실행 → 파일의 관계를 파악했는가?
- 피드백이 어떻게 시스템을 개선하는지 이해하는가?

**제출 형식**:
- 6계층 다이어그램 (Mermaid)
- Stage별 요약 문서 (Markdown)
- 파일 시스템 매핑 표
- 피드백 루프 분석 (1페이지)

**예상 소요 시간**: 2-3시간

---

#### 실습 2: 유사 프로젝트 설계

**목표**: 자신의 업무에 사례 1 패턴을 적용합니다.

**과제**:
1. 자신의 프로젝트 1개를 선택하세요 (사례 1 수준)
2. 6계층을 전부 설계하세요:
   - 1층(미션): 이 프로젝트의 존재 이유는?
   - 2층(핵심 가치): 3-4개 정의 (우선순위 포함)
   - 3층(비전): 완성된 미래 상태는?
   - 4층(목표): 측정 가능한 목표는?
   - 5층(사고 클러스터): 4단계 사고 설계
   - 6층(파일 시스템): 디렉토리 구조 설계
3. 각 Stage에 대해 상세 설계:
   - 사고 유형 (발산/수렴/생성/비판)
   - 활동 목록
   - 사용할 도구
   - 예상 산출물
4. 전체 타임라인 작성:
   - 각 Stage의 소요 시간
   - 총 예상 시간
5. thinking_state.json과 thinking_record.json 템플릿 작성

**평가 기준**:
- 6계층이 일관되게 연결되는가?
- 핵심 가치가 의사결정 기준으로 사용 가능한가?
- 4단계 사고 설계가 구체적인가?
- 파일 구조가 13장 원칙에 부합하는가?

**제출 형식**:
- 6계층 전체 설계 (YAML 또는 JSON)
- Stage별 상세 설계
- 타임라인 (Gantt 차트)
- thinking_state.json 템플릿
- thinking_record.json 템플릿 (4개 Stage)

**예상 소요 시간**: 4-6시간

---

#### 실습 3: 타임라인 및 리소스 계획

**목표**: 실행 가능한 타임라인과 리소스 배분을 계획합니다.

**과제**:
1. 실습 2의 프로젝트 사용
2. 각 Stage의 소요 시간을 추정하세요:
   - 낙관적 시나리오 (Best Case)
   - 현실적 시나리오 (Likely Case)
   - 비관적 시나리오 (Worst Case)
   - 3점 추정 평균 계산
3. 리소스를 배분하세요:
   - 인력: 누가 어느 Stage를 담당하는가
   - 도구: 각 Stage에 필요한 도구
   - 예산: 외부 도구나 서비스 비용
4. 병목 지점을 식별하세요:
   - 어느 Stage에서 지연 가능성이 높은가
   - 병목 완화 방안은?
5. 리스크를 평가하세요:
   - 발생 가능한 리스크 5개
   - 각 리스크의 대응 방안

**평가 기준**:
- 시간 추정이 현실적인가?
- 리소스 배분이 균형적인가?
- 병목 지점 대응 방안이 구체적인가?
- 리스크 평가가 충분한가?

**제출 형식**:
- Gantt 차트 (3점 추정 포함)
- 리소스 배분 표
- 병목 분석 보고서
- 리스크 관리 계획

**예상 소요 시간**: 2시간

---

### 심화 실습

#### 실습 4: 완전한 워크플로우 실행

**목표**: 설계한 프로젝트를 실제로 실행하고 결과를 측정합니다.

**과제**:
Week 1: Stage 1-2 실행
- Planning Stage 완료
  - thinking_record.json 작성
  - ideas.json 산출
- Reasoning Stage 완료
  - 핵심 가치 기반 평가
  - selected.json 산출
- 중간 검토 미팅 (30분)

Week 2: Stage 3-4 실행
- Experimenting Stage 완료
  - 초안 작성 (draft_v1.md, draft_v2.md)
  - 최종 버전 선택
- Reflecting Stage 완료
  - 품질 체크리스트 검증
  - final.md 산출
  - insights.json 작성

Week 3: 결과 측정 및 피드백
- 목표 달성 여부 평가
- 성과 데이터 수집
- 피드백 분석
- 개선 사항 도출

Week 4: 회고 및 문서화
- 전체 프로세스 회고
- 성공 요인 및 실패 요인
- 다음 프로젝트 적용 사항
- 최종 보고서 작성

**평가 기준**:
- 6계층 전체가 작동하는가?
- 모든 Stage를 완료했는가?
- thinking_state.json이 실시간 업데이트되었는가?
- 피드백 루프가 실행되었는가?
- 개선 사항이 구체적인가?

**제출 형식**:
- 완성된 파일 시스템 (모든 thinking_record)
- 최종 산출물
- 성과 측정 보고서
- 피드백 분석 문서
- 최종 회고 보고서 (5-10페이지)

**예상 소요 시간**: 2-4주

#### 실습 5: 피드백 루프 전체 사이클

**목표**: 6층 → 5층 → 4층 → 2층 피드백 루프를 완전히 구현합니다.

**과제**:

**Cycle 1 (Week 1-2): 초기 실행**
- 실습 4의 워크플로우를 실행하세요
- 결과 데이터를 체계적으로 수집하세요:
  - 목표 달성도
  - 각 Stage의 품질 점수
  - 소요 시간 (예상 vs 실제)
  - 사용자/고객 반응

**Cycle 2 (Week 3): 피드백 분석**
- 6층(실행) 분석:
  - 어떤 산출물이 좋았는가?
  - 어떤 산출물이 개선 필요한가?
- 5층(사고) 분석:
  - 어떤 Stage가 효과적이었는가?
  - 어떤 Stage에서 어려움이 있었는가?
- 4층(목표) 분석:
  - 목표가 적절했는가?
  - 목표 수정이 필요한가?
- 2층(핵심 가치) 분석:
  - 핵심 가치가 실제로 작동했는가?
  - 핵심 가치 우선순위 조정 필요한가?

**Cycle 3 (Week 4-5): 개선 적용 및 재실행**
- 개선 사항을 적용하세요:
  - Stage 프로세스 개선
  - 템플릿 업데이트
  - 체크리스트 강화
- 동일한 유형의 새 프로젝트 실행
- 개선 효과 측정

**Cycle 4 (Week 6): 효과 측정**
- 개선 전후 비교:
  - 품질 점수 비교
  - 소요 시간 비교
  - 목표 달성도 비교
- 2차 핵심 가치 정렬 검증
- 최종 학습 사항 정리

**평가 기준**:
- 4개 계층 모두 분석했는가?
- 개선 사항이 구체적으로 적용되었는가?
- 2회 실행 결과를 정량적으로 비교했는가?
- 피드백 루프가 시스템을 실제로 개선했는가?

**제출 형식**:
- Cycle 1 실행 결과
- Cycle 2 피드백 분석 보고서 (10페이지)
- Cycle 3 개선 적용 계획 + 재실행 결과
- Cycle 4 효과 측정 보고서 (Before/After)
- 전체 학습 사항 요약 (3페이지)

**예상 소요 시간**: 6-8주

#### 실습 6: 다중 사례 비교 연구

**목표**: 14.1, 14.2, 14.3의 3가지 사례를 비교 분석하여 패턴을 발견합니다.

**과제**:
1. 3가지 사례를 모두 정독하세요 (3-4시간)
2. 공통 패턴을 찾으세요:
   - 모든 사례에서 반복되는 구조는?
   - 효과적인 사고 분해 방법은?
   - 파일 구조의 공통점은?
3. 차이점을 분석하세요:
   - 사례별 독특한 특징은?
   - 복잡도에 따른 변형은?
   - 적용 난이도 차이는?
4. 자신만의 템플릿을 구축하세요:
   - 3가지 사례에서 학습한 베스트 프랙티스
   - 자신의 업무에 맞춘 커스터마이징
   - 재사용 가능한 템플릿화
5. 적용 가이드를 작성하세요:
   - 어떤 프로젝트에 어느 사례를 적용할까?
   - 사례 선택 의사결정 트리
   - 체크리스트

**평가 기준**:
- 공통 패턴을 정확히 파악했는가?
- 차이점 분석이 통찰력 있는가?
- 커스텀 템플릿이 실용적인가?
- 적용 가이드가 명확한가?

**제출 형식**:
- 사례 비교 분석 보고서 (15-20페이지)
  - 공통 패턴 (3-5개)
  - 차이점 분석
  - 사례별 장단점
- 커스텀 워크플로우 템플릿 (YAML/JSON)
- 적용 가이드 문서
  - 의사결정 트리
  - 체크리스트
  - 예시

**예상 소요 시간**: 1주

### 자가 평가

다음 질문에 답하며 이해도를 점검하세요:

1. **6계층 구조가 어떻게 통합되는가?**
   - 1층(미션) → 2층(핵심 가치) → ... → 6층(파일 시스템)
   - 핵심 가치가 모든 계층의 의사결정 기준

2. **사고 클러스터 → 파일 시스템 변환 과정은?**
   - Stage별 thinking_record.json
   - thinking_state.json으로 진행 추적
   - 산출물을 파일로 저장

3. **피드백 루프가 각 계층에 미치는 영향은?**
   - 6층 → 5층: 사고 프로세스 개선
   - 5층 → 4층: 목표 조정
   - 4층 → 2층: 핵심 가치 재확인

4. **타임라인 계획 시 고려할 핵심 요소는?**
   - 3점 추정 (낙관/현실/비관)
   - 병목 지점 식별
   - 리스크 평가

5. **워크플로우 설계의 핵심 원칙은?**
   - 지금 필요한 최소한의 복잡도
   - 6계층 전체를 항상 설계
   - 핵심 가치를 모든 결정에 적용
   - 피드백 루프 필수 구현

**적용 수준 체크**:
- [ ] 레벨 1 (기초): 사례를 분석하고 이해 가능
- [ ] 레벨 2 (중급): 자신의 프로젝트를 6계층으로 설계 가능
- [ ] 레벨 3 (고급): 설계를 실제로 실행하고 결과 측정 가능
- [ ] 레벨 4 (전문가): 피드백 루프를 운영하고 지속적 개선 가능
