# 2권 서문: 사고 중심 조직 설계

## 개인에서 조직으로

1권에서 우리는 개인의 AI 협업 여정을 따라왔습니다. 단일 프롬프트에서 시작하여 단일 에이전트를 만들어 업무를 자동화하는 방법을 배웠습니다. 이제 2권에서는 그 다음 단계로 나아갑니다. 여러 AI 에이전트가 함께 일하는 조직을 설계하는 것입니다.

하지만 이 책은 단순히 AI 에이전트를 많이 만드는 방법을 알려주지 않습니다. 그보다 더 본질적인 질문에 답합니다: **AI 시대의 조직은 어떻게 사고해야 하는가?**

## AI 사고 생태계: 6계층 구조

조직을 설계하기 전에, 먼저 조직이 어떻게 사고하고 행동하는지 이해해야 합니다. 이를 위해 우리는 **AI 사고 생태계 모델**을 소개합니다. 이 모델은 조직의 사고와 실행을 6개의 계층으로 구분합니다:

```
┌─────────────────────────────────────────┐
│  1. 미션 (Mission)                       │  ← 철학적 기반
│     "우리는 왜 존재하는가?"               │
├─────────────────────────────────────────┤
│  2. 핵심 가치 (Core Values)              │  ← 의사결정 기준
│     "우리가 지키는 원칙은 무엇인가?"      │
├─────────────────────────────────────────┤
│  3. 비전 (Vision)                        │  ← 미래 상태
│     "우리가 만들 세상은 어떤 모습인가?"   │
├─────────────────────────────────────────┤
│  4. 목표 (Goals)                         │  ← 전략적 방향
│     "비전을 실현하기 위한 구체적 목표"    │
├─────────────────────────────────────────┤
│  5. 사고 클러스터 (Thinking Clusters)    │  ← 사고와 추론
│     "복잡한 문제를 어떻게 해결할 것인가?" │
├─────────────────────────────────────────┤
│  6. 실행 항목 (Execution Items)          │  ← 구체적 행동
│     "당장 해야 할 작업은 무엇인가?"       │
└─────────────────────────────────────────┘

전략 계층 (상위 3계층): 인간이 주도
실행 계층 (하위 3계층): 인간과 AI 협업
```

### 계층별 의미

**상위 계층 (전략)**: 미션, 핵심 가치, 비전
- 조직의 철학과 방향성을 정의합니다
- 주로 인간이 설정하고 유지합니다
- 변화가 느리고 안정적입니다

**중간 계층 (실행 설계)**: 목표, 사고 클러스터
- 전략을 실행 가능한 계획으로 전환합니다
- 인간과 AI가 함께 설계합니다
- **이 책의 핵심 초점입니다**

**하위 계층 (실행)**: 실행 항목
- 구체적인 작업을 수행합니다
- AI가 반복적 작업을 담당하고, 인간이 맥락적 판단을 제공합니다

## 2권의 목적: 사고 중심 조직 설계

이 6계층 구조에서 2권은 특히 **중간 계층**, 즉 목표와 사고 클러스터 계층에 집중합니다. 왜냐하면 이것이 AI 시대 조직의 핵심 경쟁력이기 때문입니다.

전통적인 조직은 **실행 중심**이었습니다. 누가 무엇을 언제까지 할 것인가에 초점을 맞췄습니다. 하지만 AI 시대의 조직은 **사고 중심**이어야 합니다:

- **실행 중심 조직**: "이 작업을 더 빨리 처리하려면?"
- **사고 중심 조직**: "이 문제를 어떻게 해결할 것인가?"

실행은 AI가 점점 더 잘하게 됩니다. 하지만 복잡한 문제를 분해하고, 다양한 관점에서 추론하며, 맥락을 고려하여 판단하는 것은 여전히 인간과 AI가 함께해야 하는 영역입니다.

**사고 클러스터**는 바로 이 영역의 핵심 단위입니다. 복잡한 목표를 관리 가능한 사고 단위로 분해하고, 인간과 AI가 각자의 강점을 살려 협업하는 공간입니다.

## 인간과 AI의 역할

AI 사고 생태계에서 인간과 AI의 역할은 명확히 구분됩니다:

### 인간의 역할 (Human-in-the-Loop)

**전략 계층 (상위 3계층)**:
- 미션과 핵심 가치를 정의합니다
- 비전을 설정하고 전략적 목표를 세웁니다

**실행 계층 (중간 및 하위)**:
- 사고 클러스터를 설계하고 구조화합니다
- 맥락적 판단이 필요한 의사결정을 내립니다
- 피드백을 해석하고 전략을 조정합니다

### AI의 역할 (AI-in-the-Loop)

**사고 클러스터 계층**:
- 복잡한 추론과 분석을 수행합니다
- 대안을 생성하고 평가합니다
- 목표 간 충돌을 탐지합니다 (상충, 중복, 의존)

**실행 항목 계층**:
- 반복적이고 규칙적인 작업을 처리합니다
- 데이터를 수집하고 패턴을 찾습니다
- 실행 결과를 피드백으로 전달합니다

## 피드백 루프: 학습하는 조직

AI 사고 생태계의 가장 중요한 특징은 **피드백 루프**입니다. 실행 항목에서 수집된 결과는 다시 사고 클러스터로 전달되고, 필요하면 목표와 비전까지 영향을 미칩니다:

```
미션 → 핵심 가치 → 비전 → 목표
                          ↓
              사고 클러스터 ← 피드백
                          ↓
                    실행 항목
```

이러한 피드백 루프를 통해 조직은:
- 실행 결과에서 학습합니다
- 전략을 지속적으로 개선합니다
- 변화하는 환경에 적응합니다

## 2권의 구성

2권은 이 프레임워크를 바탕으로 다음과 같이 구성됩니다:

### Part 4: 사고 클러스터 설계

**11장: 기본 사고 클러스터** (목표 계층 → 사고 클러스터 계층)
- 단일 목표를 사고 클러스터로 전환하기
- 사고 조율자와 실행 워커의 역할
- 아키텍트-워커 패턴

**12장: 계층적 사고 클러스터** (복합 사고 클러스터)
- 여러 사고 클러스터를 조율하기
- 메타 조율자의 역할
- 계층적 조직 설계

### Part 5: 실행 지원과 지속적 학습

**13장: 실행 항목 지원 도구** (실행 항목 계층)
- 웹 검색, 코드 실행, API 호출
- 도구를 통한 실행 항목 강화

**14장: 사고 워크플로우 정의** (사고 클러스터 설계)
- DSL을 통한 워크플로우 명세
- 검증 가능한 사고 프로세스

**15장: 피드백 루프와 지속적 학습** (전체 계층 연결)
- 실행 → 사고 → 전략으로의 피드백
- 목표 충돌 해결 (상충, 중복, 의존)
- 학습하는 조직 만들기

## 이 책을 읽는 방법

2권은 1권과 연결되지만 독립적으로 읽을 수도 있습니다:

### 1권을 읽은 독자라면

**축하합니다!** 1권에서 배운 개념들이 2권에서 자연스럽게 확장됩니다.

**1권에서 배운 것**:
- Part 1: 프롬프트와 인스트럭션 (Ch 1-3)
- Part 2: 설계 원칙과 구성 요소 (Ch 4-6)
- Part 3: 워크플로우와 평가 (Ch 7-9)
- Part 4: 실전 패턴 (Ch 10)

**2권에서 확장되는 것**:
```
단일 프롬프트 → 인스트럭션 (1권)
    ↓
목표 → 사고 클러스터 (2권)

단일 에이전트 역할 (1권 Ch 5)
    ↓
사고 조율자 + 실행 워커 (2권 Ch 11)

기본 워크플로우 (1권 Ch 7)
    ↓
사고 워크플로우 + 피드백 루프 (2권 Ch 14-15)
```

**학습 팁**:
- 1권 개념이 낯설게 느껴지면 해당 챕터를 다시 참조하세요
- 2권의 "사고 클러스터"는 1권의 "에이전트"를 조직 수준으로 확장한 것입니다
- 전환 가이드([`transition-guide.md`](../transition-guide.md))를 참고하세요

### 2권부터 시작하는 독자라면

**조건부 권장**: AI 조직 설계에 직접 관심이 있고, 기본 개념에 익숙하다면 2권부터 시작 가능합니다.

**필수 사전 지식**:
- 프롬프트와 인스트럭션의 차이
- 에이전트 역할 개념
- 기본 워크플로우 패턴

**학습 전략**:
1. 이 서문을 꼼꼼히 읽으세요 (특히 6계층 모델)
2. Ch 11에서 막히면 1권 Ch 5(에이전트)와 Ch 7(워크플로우) 참조
3. 필요시 1권의 해당 챕터로 돌아가세요

**1권 참조가 필요한 순간**:
- "단일 에이전트"라는 용어가 나올 때 → 1권 Ch 5
- "워크플로우 패턴"이 나올 때 → 1권 Ch 7
- "평가 지표"가 나올 때 → 1권 Ch 9

**중요한 것**: 이 책은 기술적 구현보다 **사고의 구조**에 집중합니다. 코드나 특정 도구는 예시일 뿐이며, 핵심은 AI 시대 조직이 어떻게 사고하고 학습해야 하는가에 대한 프레임워크입니다.
