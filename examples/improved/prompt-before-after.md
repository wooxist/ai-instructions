# 개선 예시: 나쁜 → 좋은 지시

## Before (나쁜 지시)
"보고서 좀 잘 써줘"

## After (좋은 지시)
- 목적: 경영진 브리핑용 1페이지 요약 보고서
- 역할: 데이터 분석가
- 입력: 매출.csv, 고객리뷰.json
- 처리: 데이터 요약 → 인사이트 3개 → 리스크 2개
- 출력: Markdown 1페이지, 표 1개 포함
- 제약: 사실 기반, 과장 금지, 500단어 미만
- 성공 기준: 필수 필드 포함, 500단어 이내, 인사이트·리스크 충족

## 변화 포인트
- 모호어 제거, 형식·제약·성공 기준 명확화
